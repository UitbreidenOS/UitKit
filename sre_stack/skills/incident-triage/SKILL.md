# Incident Triage

## When to activate
Alert received from monitoring system or manual incident report.

## When NOT to use
For planned maintenance or scheduled events.

## Instructions
1. Assess severity (low/medium/high/critical)
2. Identify affected services and users
3. Estimate impact scope and duration
4. Check if runbook exists
5. Page on-call engineer if critical
6. Post initial status update

## Example
Alert: API response time > 5s. Triage: high severity, 15% of users affected, may need database scaling.

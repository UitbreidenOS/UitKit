# 📂 Devrel Workspace
> The canonical workspace for a Developer Advocate or DevRel Team, designed to scale community engagement, automate technical content distribution, and systematically track Developer Experience (DX) friction.

📄 `devrel-charter.md`           # Canonical brief: Target developer personas, core community values, and primary OKRs (e.g., active contributors vs. top-of-funnel awareness)
🧠 `active-campaigns-memory.md`  # Session memory: Dynamic context tracking for current hackathons, active CFPs, and beta-launch feedback
🤖 `CLAUDE.md`                   # Operating rules: Strict instructions to maintain an authentic, developer-first tone (strictly ban marketing fluff, prioritize code snippets and technical accuracy)

## 📁 content-and-tutorials/ (4 skills - Education)
📄 `technical-blog-drafter.md`   # Framework for writing deep-dive tutorials focusing on architecture, edge cases, and "how it works under the hood"
📄 `cross-platform-publisher.md` # Automated formatting rules to syndicate a single markdown file natively to Dev.to, Hashnode, Medium, and personal blogs
📄 `video-script-engine.md`      # Shot-lists and screen-recording outlines for 5-minute YouTube technical demos and shorts
📄 `demo-repo-scaffolder.md`     # Scripts to quickly spin up boilerplate "Hello World" GitHub repositories for live conference coding

## 📁 community-and-events/ (4 skills - Engagement)
📄 `cfp-abstract-generator.md`   # Drafts compelling Call for Papers (CFP) submissions tailored to specific tech conference themes
📄 `event-logistics-tracker.md`  # Playbook for organizing meetups, managing swag inventory, and coordinating speaker travel
📄 `ambassador-program.md`       # Guidelines for identifying, rewarding, and retaining top community champions and open-source contributors
📄 `discord-moderation-bot.md`   # Logic for automated server onboarding, spam filtering, and routing technical questions to the right channels

## 📁 developer-experience-dx/ (3 skills - Product Feedback)
📄 `api-friction-logger.md`      # Systematic template for documenting exact moments where developers get stuck during SDK onboarding
📄 `feedback-triage-router.md`   # Converts community complaints in Slack/Discord into formatted, actionable Jira tickets for the engineering team
📄 `docs-teardown-auditor.md`    # Periodic review checklist to ensure 'Getting Started' guides actually work with the latest major version release

## 📁 open-source-ops/ (3 skills - Ecosystem)
📄 `good-first-issue-tagger.md`  # Criteria for identifying and scoping approachable GitHub issues to convert lurkers into contributors
📄 `release-notes-translator.md` # Turns dry Git commit histories into exciting, community-focused release announcements
📄 `contributor-shoutouts.md`    # Automated aggregation of non-employee merged PRs for the weekly newsletter

## 📁 metrics-and-impact/ (3 skills - ROI Tracking)
📄 `utm-tag-generator.md`        # Standardizes tracking links across all blog posts, videos, and QR codes for downstream pipeline attribution
📄 `github-velocity-tracker.md`  # Monitors star growth, fork ratios, and issue resolution times to gauge project health
📄 `github-final-sync.md`        # CI/CD action to commit finalized technical blogs, event schedules, and demo code to Github final repos

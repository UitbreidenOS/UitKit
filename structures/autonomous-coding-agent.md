# 📂 Autonomous Coding Agent Workspace
> The canonical workspace for deploying and managing production-grade autonomous coding agents, designed to run continuous overnight development loops and seamlessly push validated code to your final repositories.

📄 `agent-architecture-brief.md` # Canonical brief: Defines the agent permissions, approved tech stack, and human-in-the-loop (HITL) intervention thresholds
🧠 `active-nightly-builds.md`    # Session memory: Dynamic context tracking for current overnight refactoring tasks and unresolved compile errors
🤖 `CLAUDE.md`                   # Operating rules: Strict instructions to prevent infinite execution loops and mandate deterministic API routing

## 📁 agent-orchestration/ (4 skills - The Brain)
📄 `zaltaclaw-execution-loop.md` # The core autonomous engine allowing the ZaltaClaw agent to write, test, and debug code overnight without human intervention
📄 `claude-managed-agents.md`    # Integration configurations for utilizing Claude's public beta managed agents for specialized reasoning tasks
📄 `claude-code-integration.md`  # CLI wrappers enabling the agent to execute shell commands and modify local files securely
📄 `multi-agent-coordinator.md`  # Supervisor logic for routing complex tasks across multi-agent systems hosted on AWS Bedrock

## 📁 compute-and-infrastructure/ (3 skills - Execution Environment)
📄 `mac-mini-host-config.md`     # Setup scripts for deploying the primary autonomous loop on a dedicated Mac mini to minimize cloud inference costs
📄 `aws-bedrock-allocator.md`    # Terraform configurations for spinning up scalable, on-demand agent clusters for heavy RAG pipelines
📄 `sandbox-container-rules.md`  # Docker configurations ensuring the agent cannot accidentally wipe local system files during a hallucination

## 📁 target-repositories/ (3 skills - Code Output)
📄 `health-api-backend.md`       # Standard operating procedures for the agent when contributing to the backend Python/Node repository
📄 `health-ui-frontend.md`       # Component structures and styling guidelines for the agent when updating the frontend Next.js/React repository
📄 `github-final-sync.md`        # Automated CI/CD triggers that validate the agent's overnight work and cleanly merge it into the Github final repos

## 📁 frontend-asset-pipeline/ (3 skills - UI & Design Guardrails)
📄 `texture-and-color-guardrails.md` # Strict rules intercepting the agent's image generation tools: absolutely must preserve original painting colors during website image enhancements, and enforce hyper-realistic textures (e.g., matching a natural green coconut) while stripping out any unwanted third-party monster logos
📄 `responsive-layout-tester.md` # Playwright scripts the agent runs to verify CSS grids don't break on mobile viewports
📄 `component-storyboarder.md`   # Auto-generates Storybook entries for any new UI element the agent creates

## 📁 evals-and-telemetry/ (3 skills - Quality Control)
📄 `compile-success-tracker.md`  # Monitors the ratio of successful nightly builds vs. syntax errors generated by the agent
📄 `token-burn-alerter.md`       # Hard limits that instantly pause the ZaltaClaw loop if API token usage spikes abnormally
📄 `code-review-llm.md`          # A secondary, isolated agent that exclusively critiques and scores the primary agent's pull requests

# 📂 DevRel Workspace
> The canonical workspace for a Developer Advocate or DevRel Team, designed to scale community engagement, automate technical content distribution, and systematically track Developer Experience (DX) friction.

📄 `devrel-charter.md`      # Canonical brief: ICP developer persona, advocacy pillars, and community KPIs
🧠 `active-initiatives.md`  # Session memory: Dynamic context for current DevRel campaigns and event calendar
🤖 `CLAUDE.md`              # Operating rules: Strict instructions to write for senior developers and never over-explain basic concepts

## 📁 community-engagement/ (4 skills - build and nurture developer communities)
📄 `discord-community-ops.md`           # Automate Discord moderation, FAQ responses, and routing developer questions to experts
📄 `github-discussions-strategy.md`     # Monitor GitHub Discussions, tag relevant maintainers, and surface common integration pain points
📄 `community-event-planning.md`        # Orchestrate meetups, workshops, and hackathons with templated logistics and post-event analysis
📄 `ambassador-program.md`              # Recruit, onboard, and track community ambassadors with incentive structures and impact metrics

## 📁 technical-content/ (4 skills - produce and distribute developer-focused content)
📄 `content-calendar-management.md`     # Plan, coordinate, and publish blog posts, tutorials, and case studies across channels
📄 `code-example-curation.md`           # Extract, test, and document code patterns from internal projects and community contributions
📄 `api-documentation-review.md`        # Audit API docs for clarity, completeness, and developer usability; track DX gaps
📄 `developer-newsletter.md`            # Curate weekly/monthly technical newsletters highlighting product updates, community wins, and learning resources

## 📁 developer-experience/ (3 skills - identify and resolve friction)
📄 `friction-tracking-system.md`        # Collect, categorize, and prioritize developer feedback from support tickets, surveys, and community channels
📄 `onboarding-audit.md`                # Test and document the end-to-end developer signup, auth, and first API call experience
📄 `integration-troubleshooting.md`     # Diagnose common integration failures, document workarounds, and feed learnings to product teams

## 📁 event-strategy/ (4 skills - plan and amplify developer events)
📄 `conference-sponsorship-roi.md`      # Track conference attendance, manage booth presence, and measure pipeline impact
📄 `webinar-production.md`              # Schedule speakers, manage technical setup, handle Q&A moderation, and archive content
📄 `office-hours-scheduling.md`         # Coordinate recurring developer office hours with rotating expert speakers and recording distribution
📄 `developer-relations-reporting.md`   # Aggregate community metrics, summarize monthly DevRel impact, and identify growth opportunities

## 📁 analytics-and-feedback/ (3 skills - measure and iterate)
📄 `community-health-dashboard.md`      # Track GitHub stars, Discord growth, event attendance, and content engagement across platforms
📄 `developer-sentiment-analysis.md`    # Analyze forum posts, Twitter/X mentions, and issue comments to identify sentiment shifts
📄 `feedback-synthesis-reports.md`      # Synthesize quarterly feedback from all sources into actionable product and advocacy recommendations

---
**Configuration Files**

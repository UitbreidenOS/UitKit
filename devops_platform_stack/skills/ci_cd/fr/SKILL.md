# Pipelines CI/CD

## Quand activer

Création, débogage ou optimisation des flux de travail CI/CD dans GitHub Actions, GitLab CI ou Jenkins.

## Quand NE PAS utiliser

Pour les runbooks de déploiement manuel — utilisez plutôt les guides de déploiement.

## Instructions

1. Analyser le YAML du pipeline et identifier les goulots d'étranglement
2. Valider les dépendances des tâches et l'exécution parallèle
3. Examiner la gestion des secrets et les contrôles d'accès
4. Optimiser le cache de compilation et la gestion des artefacts

## Exemple

Réduire la durée du pipeline de 45 min à 18 min en parallélisant les tests et en mettant en cache les dépendances.

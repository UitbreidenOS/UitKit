# Onboarding Workflow

## When to activate

Immediately after offer is signed. New hire has accepted and set start date. You have access to candidate's role, department, manager, and company's standard onboarding process.

## When NOT to use

Not for offboarding (use off-boarder skill). Not for internal transfers (modify onboarding-workflow for lateral moves). Not without a confirmed start date and manager assignment.

## Onboarding Goals

- **Day 1:** Employee feels welcomed, has workspace, knows who to talk to.
- **Week 1:** Employee completes admin tasks, meets core team, understands company values and role context.
- **Week 2–4:** Employee is ramping: has access, understands codebases/processes, is shipping first contributions.
- **Month 3:** Employee is productive, has feedback from manager and peers, knows next growth areas.

## Onboarding Dimensions

### 1. Pre-Arrival Setup (1 week before start)

**What to do:**
- [ ] Send welcome email with start date, time, location, parking, building access code
- [ ] Provision laptop (macbook size preference, OS preference)
- [ ] Request system access (email, Slack, GitHub, JIRA, calendar)
- [ ] Set up desk, phone, hardware (monitor, keyboard, dock)
- [ ] Schedule first-day meeting with manager (30 min, welcome + context)
- [ ] Schedule team lunch or coffee with buddy (2h, informal intro)
- [ ] Assign onboarding buddy (peer, not manager; 1–2 levels similar)
- [ ] Create onboarding document (shared doc: org structure, key processes, team rituals)
- [ ] Add to calendar: team meetings, 1:1 with manager (recurring), standup (if applicable)

### 2. Day 1 Checklist

**8am — Arrival & Setup:**
- [ ] Welcome email sent; confirm arrival
- [ ] Laptop, monitors, phone ready on desk
- [ ] Workspace assigned and accessible
- [ ] Parking pass provided (if applicable)
- [ ] Building badge/security access working

**9am — Manager Welcome Meeting (30 min):**
- [ ] Role overview: what you'll own in first 30 days
- [ ] Team structure: who you'll work with most
- [ ] First week goals: what's a win?
- [ ] Manager availability: office hours, communication preferences
- [ ] Questions: "What do you want to learn first?"

**10am — Buddy Meet-Up (30 min):**
- [ ] Informal intro, peer perspective on team/culture
- [ ] Where to grab coffee, lunch
- [ ] Company culture tidbits (unwritten rules)
- [ ] Buddy's phone number for questions

**11am — IT Onboarding (30 min):**
- [ ] System access: email, Slack, GitHub, JIRA, calendar
- [ ] Password manager setup (1Password, LastPass, etc.)
- [ ] VPN setup (if remote)
- [ ] Slack channels to join (team, general, random, #new-starters)
- [ ] Calendar permissions and syncing

**12pm — Team Lunch:**
- [ ] Casual team lunch (1h)
- [ ] Intro to core team members
- [ ] Light conversation, no heavy topics

**2pm — Company Overview (30 min):**
- [ ] Product walkthrough: company mission, values, roadmap (1-slide)
- [ ] Org structure: teams, leadership (5 min)
- [ ] Key rituals: standup, all-hands, retro (5 min)
- [ ] Success metrics: what does good look like? (5 min)

**3pm — Self-Onboarding (1h):**
- [ ] Read: Company handbook, values, code of conduct
- [ ] Watch: Company orientation video (if exists)
- [ ] Skim: Team onboarding doc
- [ ] Explore: Codebase README, architecture overview

**4pm — Kickoff & Close:**
- [ ] Manager check-in: "How was day 1? Any blockers?"
- [ ] Set goals for week 1
- [ ] Confirm tomorrow's schedule

### 3. Week 1 Checklist

**Goals for week:**
- Understand your team's core mission and current priorities
- Set up all systems and access
- Meet 5–8 key people across the company (adjacent teams, cross-functional partners)
- Read core documentation (codebases, internal wikis, playbooks)
- Attend key meetings (standup, team meeting, all-hands)
- Submit a low-risk PR or contribution (e.g., docs update, small fix)

**Monday:**
- [ ] 1:1 with manager (30 min)
- [ ] Onboarding buddy check-in (15 min)
- [ ] Team standup (if applicable)
- [ ] Codebase exploration: README, architecture docs, main services

**Tuesday:**
- [ ] Team meeting (learn priorities, current projects)
- [ ] Meet 1–2 cross-functional partners (design, product, etc.)
- [ ] Environment setup: can you build/run code locally?
- [ ] Submit first PR (docs, small improvement)

**Wednesday:**
- [ ] All-hands meeting (company updates, values reinforcement)
- [ ] 1:1 with manager (progress, blockers)
- [ ] Code review training: how does your team review code?
- [ ] Pair with teammate on small task

**Thursday:**
- [ ] Retrospective: what went well, what didn't?
- [ ] Meet 1–2 more team members
- [ ] Work on your first "real" task (guided by manager)
- [ ] Async documentation: what was confusing? Update docs.

**Friday:**
- [ ] 1:1 with manager (week 1 retrospective, week 2 goals)
- [ ] Team social (coffee, drinks, etc.)
- [ ] Self-reflection: progress check
- [ ] Document blockers or questions for next week

### 4. Month 1 Goals

By end of week 1, employee should have:
- [ ] Systems access complete (email, Slack, GitHub, JIRA, etc.)
- [ ] First PR submitted and reviewed
- [ ] Met 5–8 key people
- [ ] Attended all core team meetings
- [ ] Read core documentation

By end of week 2, employee should be:
- [ ] Shipping small contributions (code, docs, design)
- [ ] Attending standups, participating
- [ ] Asking good questions (curiosity signal)
- [ ] Starting on first sprint task

By end of week 4, employee should be:
- [ ] Owning a small project or task area
- [ ] Contributing meaningfully to team
- [ ] Feedback from manager: on track for role success
- [ ] 30-day 1:1: how are you feeling? What's next?

## System Access Checklist

Request access to:

**Communication:**
- [ ] Email (AD/Microsoft/Google)
- [ ] Slack (join core channels)
- [ ] Zoom/video conferencing

**Development (if engineer):**
- [ ] GitHub (organization, relevant repos)
- [ ] AWS/GCP/Azure (dev environment access)
- [ ] JIRA/Linear (issue tracking, sprint board)
- [ ] Datadog/monitoring (observability tools)
- [ ] Figma (if design/product)
- [ ] Notion/wiki (internal documentation)

**HR & Admin:**
- [ ] Workday/BambooHR (benefits, payroll)
- [ ] 401(k) provider (retirement account)
- [ ] Insurance portals (health, dental, vision)
- [ ] Equity portal (stock option documents)
- [ ] Expensify (expense reporting)

**Other:**
- [ ] Building access badge
- [ ] Parking pass
- [ ] Phone (corporate, if applicable)
- [ ] Software licenses (IDEs, tools)

## Buddy Assignment Framework

**Buddy criteria:**
- Peer level (not manager, not entry-level if hire is senior)
- 1–2 years tenure (experienced enough to guide, not too senior)
- Friendly, patient demeanor
- Works on adjacent or same team (familiar with context)
- Available for quick questions first 2 weeks (not on vacation)

**Buddy responsibilities:**
- Day 1: meet new hire, show around, grab lunch
- Week 1: available for quick questions (Slack, coffee)
- Week 2–4: pair on tasks, answer context questions, social intro
- Week 5+: check in async, transition to regular teammate dynamic

**Buddy is NOT:**
- The manager (manager does performance feedback)
- HR (HR does compliance, benefits)
- The new hire's primary stakeholder (manager is)

# Employee Onboarding Coordinator

## When to activate
After hiring approval and offer acceptance, before or on employee start date. Use to create structured onboarding that accelerates time-to-productivity and reduces first-week friction.

## When NOT to use
Skip if the employee's role already has a fully documented onboarding playbook. Skip for contractors or temporary placements. Skip if employee has previously worked at your organization.

## Onboarding Framework

**Pre-Arrival (Week before)**
- Manager welcome email with start date, office logistics, parking, WiFi setup
- IT provisioning: laptop, email, accounts, software licenses
- Peer assignment (buddy or mentor)
- Desk/equipment setup assigned
- Welcome package: org chart, handbook, team directory

**Day One**
- Reception and building access
- IT credential handoff and tech orientation
- 1:1 with direct manager: role expectations, team structure, success metrics
- Team meet-and-greet (lunch or coffee)
- Office tour and access to restrooms, kitchen, parking
- Benefits/payroll orientation overview

**Week One**
- Role-specific training (tools, systems, processes)
- Cross-functional introductions (key stakeholders, adjacent teams)
- Understand reporting lines and escalation paths
- Initial project assignment (low-risk, collaborative)
- Welcome survey: capture feelings, identify blockers

**Month One**
- 30-day 1:1 debrief with manager
- Department or all-hands meeting participation
- Completion of mandatory compliance/safety training
- First feedback loop: what's working, what needs adjustment
- Social integration: team lunch, after-work event
- Clear assignment to first real project

## Key Checkpoints

1. **Accountability:** Manager owns pre-arrival; HR coordinates day-one logistics; L&D handles training rollout
2. **Flexibility:** Build in time for questions and surprises—don't over-schedule
3. **Culture immersion:** Mix formal training with informal social touchpoints
4. **Documentation:** Log all completed activities and employee feedback to session log
5. **Feedback loop:** Capture onboarding sentiment on day 1, day 7, and day 30

## Example

**Employee:** Alex Kumar, Senior Software Engineer, starts 2026-07-15

```
ONBOARDING PLAN: Alex Kumar · Senior Software Engineer · 2026-07-15

PRE-ARRIVAL CHECKLIST (Due: 2026-07-08)
□ Welcome email: "Looking forward to you joining the Platform team on July 15. You'll be working on our API infrastructure alongside Sam and Jamie."
□ Laptop ordered (MacBook Pro M3), email created (alex.kumar@company.com)
□ Buddy assigned: Sam Chen (Senior Engineer, 3 years tenure)
□ Standing desk reserved in West wing near platform team
□ Welcome package prepped: handbook, org chart, AWS account setup guide

DAY ONE SCHEDULE (2026-07-15)
09:00 — Reception at front desk, building tour
09:30 — IT setup: laptop, phone, badges, VPN
10:30 — 1:1 with hiring manager (Sarah): role scope, team structure, 90-day goals
12:00 — Lunch with Sam (buddy) and two peers from platform team
14:00 — Office tour, code of conduct acknowledgment
15:30 — Benefits and payroll intro (HR)

WEEK ONE LEARNING PLAN
□ Day 1–2: Onboarding module + platform architecture overview video
□ Day 2: GitHub/CI-CD setup; local dev environment walk-through with Sam
□ Day 3: Cross-functional intros: (Product, DevOps, QA leads)
□ Day 4: First code review pairing session
□ Day 5: Sprint planning; assigned to refactoring ticket (low-risk, collaborative)

MONTH ONE MILESTONES
□ 1:1 with Sarah (day 28): progress check, wins, blockers
□ Mandatory training completed: security, harassment prevention, code of conduct
□ First refactoring ticket shipped to production
□ Attended 2+ social events (team lunch, engineer happy hour)

SUCCESS METRICS
- Alex completes all Day 1 activities ✓
- Local dev environment working by end of Day 2
- First code review merged by end of Week 2
- Feels welcomed and productive by 30-day mark (survey)
```

# Habilidad Codebase Onboarding

## Cuándo activar
- Preparar base de código para nuevos contribuidores
- Documentar arquitectura del sistema
- Crear guías de inicio rápido
- Establecer convenciones de código
- Configurar proceso de contribución

## Cuándo NO usar
- Documentación API — use api-docs skill
- Escribir código — use development skills

## Instrucciones

```
Onboarding para [proyecto].

Tipo: [monolith / microservices / serverless / data pipeline]
Equipo: [X desarrolladores, niveles: junior/mid/senior]
Stack: [tech stack del proyecto]

Documentación requerida:

1. README.md
   - Qué es
   - Configuración local
   - Ejecutar pruebas
   - Arquitectura de alto nivel (1-2 diagramas)

2. ARCHITECTURE.md
   - Diagrama de componentes
   - Flujo de datos
   - Puntos de integración externa
   - Decisiones arquitectónicas clave

3. CONTRIBUTING.md
   - Proceso de contribución
   - Convenciones de código
   - Proceso de revisión
   - Pruebas requeridas

4. TROUBLESHOOTING.md
   - Problemas comunes
   - Soluciones rápidas
   - Dónde pedir ayuda

Genere documentación completa para mi base de código.
```

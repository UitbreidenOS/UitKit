# Habilidad Dependency Auditor

## Cuándo activar
- Escanee dependencias para vulnerabilidades
- Genere SBOM
- Planifique upgrades de dependencias
- Gestione vulnerabilidades encontradas
- Auditoría de cumplimiento

## Instrucciones

```
Auditoría de dependencias para [proyecto].

Lenguaje: [npm / pip / Maven / Go / Cargo]
Alcance: [producción / todos]

Herramientas: npm audit, pip audit, Snyk, Dependabot, FOSSA

Proceso:
1. Escanee estado actual
2. Categorice por severidad (Critical/High/Medium/Low)
3. Evalúe impacto de upgrades
4. Planifique upgrades
5. Pruebe upgrades
6. Despliegue

SBOM (Software Bill of Materials):
- Genere con: syft, cyclonedx, o herramientas nativas del lenguaje
- Formato: CycloneDX JSON/XML o SPDX

Genere reporte de auditoría y plan de remediación para mi proyecto.
```

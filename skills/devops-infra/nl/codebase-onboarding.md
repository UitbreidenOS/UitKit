# Vaardigheid Codebase Onboarding

## Wanneer activeren
- Codebase voor nieuwe contributors voorbereiden
- Systeemarchitectuur documenteren
- Quick-start-gidsen maken
- Code-conventies vaststellen
- Contributions-proces configureren

## Wanneer NIET gebruiken
- API-documentatie — use api-docs skill
- Code schrijven — use development skills

## Instructies

```
Onboarding voor [project].

Type: [monolith / microservices / serverless / data pipeline]
Team: [X developers, niveaus: junior/mid/senior]
Stack: [tech stack van project]

Vereiste documentatie:

1. README.md
   - Wat is het
   - Lokale setup
   - Tests uitvoeren
   - High-level architectuur (1-2 diagrammen)

2. ARCHITECTURE.md
   - Component-diagram
   - Gegevensstroom
   - Externe integratiepoints
   - Belangrijkste architectuurbeslissingen

3. CONTRIBUTING.md
   - Contributions-proces
   - Code-conventies
   - Review-proces
   - Vereiste tests

4. TROUBLESHOOTING.md
   - Veelvoorkomende problemen
   - Quick fixes
   - Waar om hulp vragen

Volledige documentatie voor mijn codebase genereren.
```

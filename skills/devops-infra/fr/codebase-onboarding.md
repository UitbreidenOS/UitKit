# Compétence Onboarding Base de Code

## Quand l'activer
- Préparer une base de code pour les nouveaux contributeurs
- Documenter l'architecture du système
- Créer des guides de démarrage rapide
- Établir les conventions de code
- Configurer le processus de contribution

## Quand NE PAS l'utiliser
- Documentation API — use api-docs skill
- Rédaction de code — use development skills

## Instructions

```
Onboarding pour [projet].

Type: [monolith / microservices / serverless / data pipeline]
Équipe: [X développeurs, niveaux: junior/mid/senior]
Stack: [tech stack du projet]

Documentation requise:

1. README.md
   - Qu'est-ce que c'est
   - Comment configurer localement
   - Comment exécuter les tests
   - Architecture haute niveau (1-2 diagrammes)

2. ARCHITECTURE.md
   - Diagramme de composants
   - Flux de données
   - Points d'intégration externes
   - Décisions architecturales clés

3. CONTRIBUTING.md
   - Processus de contribution
   - Conventions de code
   - Processus de révision
   - Tests requis

4. TROUBLESHOOTING.md
   - Problèmes courants
   - Solutions rapides
   - Où demander de l'aide

Générer documentation complète pour ma base de code.
```

# Fähigkeit Codebase Onboarding

## Wann aktivieren
- Codebase für neue Contributors vorbereiten
- Systemarchitektur dokumentieren
- Quick-Start-Guides erstellen
- Code-Konventionen etablieren
- Contributions-Prozess konfigurieren

## Quand NE PAS l'utiliser
- API-Dokumentation — use api-docs skill
- Code-Schreiben — use development skills

## Anweisungen

```
Onboarding für [Projekt].

Typ: [monolith / microservices / serverless / data pipeline]
Team: [X Entwickler, Level: junior/mid/senior]
Stack: [Tech-Stack des Projekts]

Erforderliche Dokumentation:

1. README.md
   - Was ist es
   - Lokales Setup
   - Tests ausführen
   - High-Level-Architektur (1-2 Diagramme)

2. ARCHITECTURE.md
   - Komponenten-Diagramm
   - Datenfluss
   - Externe Integrations-Punkte
   - Wichtige Architektur-Entscheidungen

3. CONTRIBUTING.md
   - Contributions-Prozess
   - Code-Konventionen
   - Review-Prozess
   - Erforderliche Tests

4. TROUBLESHOOTING.md
   - Häufige Probleme
   - Quick Fixes
   - Wo um Hilfe fragen

Komplette Dokumentation für meine Codebase generieren.
```

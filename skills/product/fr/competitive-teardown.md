# Compétence Démantèlement Concurrentiel

## Quand l'activer
- Analyser un concurrent spécifique avant une décision produit
- Identifier le paysage concurrentiel de sa catégorie à partir de zéro
- Créer une battle card pour les ventes
- Trouver des espaces blancs mal desservis dans un marché encombré
- Évaluer un nouveau marché avant d'y entrer
- Préparer une présentation pour le conseil ou les investisseurs sur la concurrence

## Quand ne pas l'utiliser
- Recherche de tarification uniquement — utiliser la compétence pricing-strategy
- Analyse win/loss des clients (nécessite des données réelles d'affaires) — ceci est une intelligence concurrentielle qualitative
- Benchmarking concurrentiel juridique — utiliser la compétence legal
- Surveillance en temps réel de l'activité concurrente — utiliser la recherche web directement

## Instructions

### Démantèlement complet d'un concurrent

```
Faites un démantèlement structuré de [concurrent].

Concurrent : [nom + URL]
Notre produit : [nom + URL]
Pourquoi ce concurrent compte : [concurrent direct / adjacent / aspirationnel]

Dimensions du démantèlement :

1. Positionnement et messaging :
   - Que réclament-ils sur leur page d'accueil hero ?
   - Qui disent-ils que c'est pour (ICP explicite) ?
   - Dans quelle catégorie se positionnent-ils ?
   - Quel est leur différenciateur clé ?
   - Quelle émotion dirigent-ils (peur / aspiration / confiance / efficacité) ?

2. Produit :
   - Fonctionnalité centrale (ce qu'il fait)
   - Caractéristiques clés visibles dans le produit public + documentation
   - Caractéristiques manquantes évidentes (ce qu'ils ne font pas)
   - Écosystème d'intégration (à quoi se connectent-ils)
   - Application mobile ? Application de bureau ? API uniquement ?

3. Tarification :
   - Modèle (forfaitaire / par siège / basé sur l'utilisation / freemium / entreprise uniquement)
   - Niveaux et prix (si publics)
   - Essai gratuit ou freemium disponible ?
   - Qu'est-ce qui est derrière le paywall ?

4. Go-to-market :
   - Canal d'acquisition principal (SEO / payant / PLG / ventes dirigées / communauté)
   - Stratégie de contenu (sujets de blog, présence YouTube, podcast)
   - Partenariats et intégrations comme distribution
   - Modèle de vente (self-serve / inside sales / field sales)
   - Taille d'entreprise cible (PME / mid-market / entreprise)

5. Forces et faiblesses :
   - 3-5 choses qu'ils font mieux que nous
   - 3-5 choses qu'ils font mal ou ne font pas du tout
   - Leur vulnérabilité de positionnement (où ils ne peuvent pas concourir de manière crédible)

6. Sentiment des clients (depuis les sites d'avis — G2, Capterra, Reddit) :
   - Plaintes principales (ce que les clients n'aiment pas)
   - Éloges principaux (ce que les clients adorent le plus)
   - Raisons courantes de changement (pourquoi les clients les quittent)

Résultat : résumé du démantèlement + 3 implications stratégiques pour notre produit ou positionnement.
```

### Carte du paysage concurrentiel

```
Cartographiez le paysage concurrentiel pour [catégorie].

Catégorie : [décrivez l'espace de problèmes — ex. « gestion de projets pour les équipes d'ingénierie »]
Notre produit : [nom]
Concurrents connus : [liste — ou « les identifier »]

Cartographie du paysage :

Étape 1 — Identifier tous les concurrents :
- Direct : résoudre le même problème pour le même client
- Indirect : résoudre le même problème différemment (ex. Excel comme concurrent de tout SaaS)
- Adjacent : résoudre un problème connexe qui chevauche le nôtre
- Émergent : nouveaux entrants ou créateurs de catégories

Étape 2 — Positionner sur deux axes (choisissez les dimensions les plus stratégiques) :
   Axe 1 : [ex. Simple ← → Puissant]
   Axe 2 : [ex. Centré PME ← → Centré Entreprise]
   
   Placez chaque concurrent sur ce 2x2.
   Trouvez : où est l'espace blanc ? Où est notre positionnement ?

Étape 3 — Dynamiques de catégorie :
   - Qui est le leader de catégorie (plus de sensibilisation) ?
   - Qui croît le plus rapidement ?
   - Qui décline ?
   - La catégorie se consolide-t-elle ou se fragmente-t-elle ?

Étape 4 — Implications stratégiques :
   - Pouvons-nous posséder un segment que le leader ignore ?
   - Existe-t-il un persona mal desservi ?
   - Existe-t-il une combinaison de caractéristiques que personne ne fait bien ensemble ?

Produisez : description de la carte du paysage + 3 opportunités de positionnement que nous pourrions posséder.
```

### Battle card

```
Construisez une battle card contre [concurrent].

Concurrent : [nom]
Notre scénario concurrentiel le plus courant : [où nous gagnons / où nous perdons]
Utilisateur cible : [à qui nous vendons, son rôle]
Objection clé de ce concurrent : [« Pourquoi ne pas simplement utiliser [concurrent] ? »]

Structure de la battle card :

1. DIFFÉRENCIATEUR EN UNE LIGNE (pour que le représentant le dise en < 10 secondes) :
   « [Notre produit] fait X, [concurrent] ne peut pas parce que [raison structurelle]. »

2. OÙ NOUS GAGNONS (3 puces — concrètes, pas génériques) :
   □ [Caractéristique ou capacité spécifique qu'ils n'ont pas] → bénéfice client
   □ [Avantage de tarification ou de modèle] → ce que le client économise ou évite
   □ [Avantage d'intégration ou de flux de travail] → flux de travail spécifique qu'ils se soucient

3. OÙ ILS GAGNENT (soyez honnête — 2 puces) :
   □ [Chose qu'ils font mieux] — « Si c'est leur priorité absolue, ils auraient raison de choisir [concurrent] »
   □ [Force que nous n'avons pas] — le reconnaître, puis le rediriger

4. COMMENT LE RETOURNER :
   Pour chaque force concurrente, le redirection :
   « [Concurrent] a [force], mais le coût est [compromis qu'ils paient]. Nos clients trouvent que [notre approche] est meilleur parce que [raison spécifique]. »

5. POINTS DE PREUVE :
   - Citation client : « [Citation exacte sur pourquoi ils nous ont choisis plutôt que le concurrent] » — [Rôle, Entreprise]
   - Histoire de victoire : [Histoire de 2 phrases d'une affaire que nous avons remportée contre eux]
   - Point de données : [stat qui montre notre avantage — vitesse, NPS, rétention, peu importe]

6. MINES À PLANTER :
   Questions que le représentant devrait poser pour révéler les faiblesses du concurrent :
   - « Quelle est l'importance de [notre force] pour vous ? »
   - « Avez-vous jamais expérimenté [faiblesse du concurrent] ? »
   - « Que se passe-t-il lorsque vous avez besoin de [chose qu'ils ne font pas] ? »

Écrivez la battle card pour mon concurrent spécifique.
```

### Analyse Win/Loss

```
Concevez un processus d'analyse win/loss pour [entreprise].

État actuel : [pas de suivi win/loss / ad hoc / systématique]
Cycle de vente : [self-serve / inside sales / entreprise]
Volume d'affaires : [X affaires/mois]

Cadre win/loss :

1. Qui interviewer :
   - Affaires gagnées : interviewer 30-60 jours après la fermeture (la lune de miel est finie, les vraies raisons émergent)
   - Affaires perdues : interviewer dans les 2 semaines (la mémoire s'efface, les concurrents avancent)
   - Interviewer l'acheteur réel, pas le champion — la rationale de l'acheteur économique diffère

2. Questions pour entrevue gagnante :
   - « Qu'est-ce qui vous a fait nous choisir plutôt que les alternatives ? »
   - « Qu'est-ce qui vous aurait presque fait choisir quelqu'un d'autre ? »
   - « Qu'est-ce qui aurait fait que vous NE nous choisiriez PAS ? »
   - « Quelle caractéristique ou moment vous a confirmé que nous avions raison ? »
   - « Que dites-vous à vos collègues quand vous nous recommandez ? »

3. Questions pour entrevue perdue :
   - « Qu'est-ce qui vous a fait choisir [concurrent / rien] à la place ? »
   - « Qu'est-ce que nous avons bien fait dans le processus ? »
   - « Était-ce le produit, la tarification, le processus ou le timing ? »
   - « Qu'est-ce qui aurait dû être différent pour que vous nous choisissiez ? »
   - « La porte est-elle ouverte à l'avenir ? Qu'est-ce qui devrait changer ? »

4. Cadence d'analyse :
   - Mensuel : taux win/loss par concurrent, par segment, par taille d'affaire
   - Trimestriel : synthèse des thèmes → lacunes produit, ajustements de positionnement
   - Annuel : examen complet du paysage

5. Format de résultat :
   - Raisons de gagner (top 3) + % de fréquence
   - Raisons de perte (top 3) + % de fréquence
   - Modèles spécifiques au concurrent
   - Carte thermique des lacunes produit (ce que les clients voulaient que nous n'avions pas)

Configurez ce processus pour ma motion de vente et la taille de mon équipe.
```

## Exemple

**Utilisateur :** Nous créons des outils internes pour développeurs. Notion vient de lancer de la documentation pour développeurs. Devrions-nous être préoccupés ? Faites un démantèlement.

**Encadrement stratégique de Claude :**

Notion entrant dans votre espace est une menace pour le bas de gamme (petites équipes, budget faible) mais crée une opportunité en haut de gamme (où Notion sera toujours « assez bon » plutôt que personnalisé).

**Contraintes structurelles de Notion (choses qu'ils ne peuvent pas changer) :**
1. Produit horizontal — ne peuvent pas approfondir sur les caractéristiques spécifiques aux développeurs (examen de code, intégration PR, documentation de déploiement) sans aliéner leurs utilisateurs non-dev
2. Éditeur basé sur blocs — excellent pour la documentation structurée, mauvais pour la référence API ou les documents auto-générés du code
3. Pas d'indexation native du code — ils ne peuvent pas extraire de votre codebase et garder les documents synchronisés automatiquement
4. L'outillage d'entreprise pour développeurs nécessite SSO/SCIM, journaux d'audit, intégration git — l'entreprise de Notion est conçue pour RH et ops, pas les équipes de plateforme

**Où vous devriez différencier immédiatement :**
- Synchronisation automatique avec le code (documentation qui se met à jour quand le code change — Notion ne peut pas le faire sans casser leur modèle)
- Formats natifs aux développeurs : rendu OpenAPI, terrain de jeu de code, changelog à partir de commits git
- Intégration IDE (VS Code, JetBrains — Notion n'a aucune place ici)

**Votre réponse de messaging :**
Ne mentionnez pas Notion par nom. Au lieu de cela :
Hero : « Documentation qui reste synchronisée avec votre code — automatiquement. »
Ceci est la chose que Notion ne peut structurellement pas offrir, encadrée comme un bénéfice (pas une comparaison).

**Verdict :** Inquiétez-vous de Notion prenant le segment PME que vous alliez acquérir. Ne vous inquiétez pas de Notion prenant vos clients existants d'entreprise ou de mid-market qui ont besoin de profondeur.

# Compétence Dossier de recherche

## Quand l'activer
- Construire un dossier de recherche sur une entreprise avant un partenariat, une acquisition ou un investissement
- Rechercher une personne avant une réunion du conseil, une décision d'embauche ou une négociation
- Cartographier un paysage concurrentiel avant d'entrer sur un marché
- Approfondir une technologie ou un framework avant une décision construire-vs-acheter
- Préparer une réunion enjeux élevés où vous avez besoin de contexte complet rapidement

## Quand NE PAS l'utiliser
- Vérification rapide de base (Google suffit pour le niveau surface)
- Revue de littérature académique — utiliser la compétence lit-review
- Paysage de brevets — utiliser la compétence patent-analysis
- Surveillance d'actualités en temps réel — utiliser un agrégateur d'actualités dédié

## Instructions

### Dossier d'entreprise

```
Construire un dossier d'entreprise sur [nom de l'entreprise].

Objectif : [cible d'acquisition / évaluation de partenariat / intelligence concurrentielle / préparation investisseur]
Profondeur : [résumé exécutif (1 page) / dossier complet (10-15 pages)]
Public : [PDG / conseil / comité d'investissement / équipe d'affaires]

Structure du dossier :

1. INSTANTANÉ DE L'ENTREPRISE (toujours inclure) :
   Fondée : [année] | Siège : [localisation] | Stade : [privée/publique, stade financement]
   Employés : [plage] | ARR/Revenu : [si connu] | Financement dernier : [$X, date, investisseur leader]

2. MODÈLE D'AFFAIRES :
   Ce qu'ils vendent : [description de produit / service en 2 phrases]
   Qui l'achète : [profil client — taille, industrie, rôle]
   Comment ils font de l'argent : [abonnement / utilisation / transaction / services]
   Métriques clés : [taux croissance ARR / NRR / CAC / LTV — si disponible de sources publiques]

3. PRODUIT :
   Capacités produit principal : [ce qu'il fait]
   Signaux pile technologique : [des offres d'emploi, open source, blog d'ingénierie]
   Différenciateurs revendiqués : [de leur marketing, études de cas]
   Limitations connues : [des avis clients, communauté, rapports analystes]

4. POSITION SUR LE MARCHÉ :
   Catégorie : [comment ils définissent le marché]
   Concurrents principaux : [lister avec brève différenciation]
   Estimations part de marché : [rapports analystes, signal communauté]
   Positionnement : [où ils se situent sur axes simple/puissant, PME/entreprise]

5. LEADERSHIP ET ÉQUIPE :
   PDG : [contexte, entreprises précédentes, années à ce poste]
   CTO : [contexte, signaux crédibilité technique]
   Investisseurs clés et membres du conseil : [avec historique]
   Signaux d'équipe : [modèles d'embauche, activité LinkedIn, embauches/départs notables]

6. SIGNAUX CLIENT ET MARCHÉ :
   Avis clients : [G2, Capterra, Reddit, sentiment Hacker News]
   Clients notables (publics) : [logos si disponibles]
   Santé communauté : [taille Slack/Discord, étoiles GitHub le cas échéant]
   Sentiment presse : [couverture récente, controverses]

7. SIGNAUX DE SANTÉ FINANCIÈRE (pour entreprises privées) :
   Autonomie de financement : [estimé de dernière levée + taux burn connu]
   Signaux revenu : [croissance effectifs, expansion bureaux, changements prix]
   Signaux levée de fonds : [offres d'emploi LinkedIn rôles finance, signaux pitch]
   Signaux détresse : [réductions effectifs, départs leadership, repli prix]

8. ÉVALUATION STRATÉGIQUE :
   Pourquoi cette entreprise compte : [signification stratégique pour l'objectif déclaré]
   Risques clés : [compétitif, réglementaire, financier, produit]
   Opportunités clés : [ce qu'ils pourraient devenir, où ils vont]
   Notre décision : [procéder / enquête supplémentaire / passer — et 2-3 facteurs décisifs]

Sources de recherche à utiliser :
Primaires : site web entreprise, page investisseur, docs produit, blog ingénierie
Secondaires : Crunchbase, LinkedIn, G2/Capterra, GitHub, offres d'emploi, communiqués presse
Communauté : Hacker News, Reddit (r/[subreddits pertinents]), mentions Twitter/X
Analystes : Gartner, Forrester, IDC (si disponible), CB Insights
```

### Dossier personnel

```
Construire un dossier personnel sur [nom personne] pour [objectif].

Objectif : [candidat conseil / embauche executive / contrepartie négociation / conférencier / conseiller]
Portée : [contexte professionnel / réputation / travail public / drapeaux rouges]

Note : Ce dossier couvre uniquement les informations professionnelles publiquement disponibles. Ne pas tenter d'accéder aux informations privées, comptes personnels, ou mener de surveillance.

Structure du dossier :

1. CONTEXTE PROFESSIONNEL :
   Rôle actuel : [titre, entreprise, depuis quand]
   Rôles précédents : [chronologique, focus sur pertinents]
   Éducation : [diplômes, établissements — si publiquement listés]
   Durée au rôle actuel : [X années — constructeur ou court mandat ?]

2. HISTORIQUE :
   Réalisations notables : [entreprises construites, produits expédiés, fonds levés, résultats]
   Métriques publiques : [si fondateur — sorties entreprise, ARR atteint, taille équipe construite]
   Échecs : [échecs entreprise publique, départs — contexte compte]
   Modèle : [entrepreneur série ? opérateur ? investisseur ? conseiller ?]

3. LEADERSHIP PENSÉE :
   Écriture publiée : [blog, Substack, Medium — quels sujets, quelle perspective]
   Conférences : [conférences, podcasts — sur quoi ils parlent]
   Présence ligne : [signal followers LinkedIn, engagement Twitter/X, activité GitHub]
   Réputation communauté : [comment référencés par pairs]

4. RÉSEAU ET RÉFÉRENCES :
   Postes conseil : [quelles entreprises, quel rôle]
   Portefeuille investissement : [si investisseur — ce qu'ils ont soutenu, sorties]
   Co-fondateurs connus ou collaborateurs proches : [modèle relations travail]
   Connexions mutuelles : [connexions mutuelles LinkedIn pouvant contextualiser]

5. DRAPEAUX ROUGES (le cas échéant) :
   Légaux : [litiges publics, actions réglementaires — de PACER, SEC EDGAR]
   Réputation : [controverses publiques, plaintes communauté, Glassdoor si fondateur/exec]
   Modèle : [mandats courts multiples, pivots fréquents sans récit clair]

6. ÉVALUATION POUR [OBJECTIF] :
   Forces pour ce rôle : [spécifique à pourquoi vous les recherchez]
   Préoccupations : [ce à sonder en conversation]
   Priorité vérification référence : [qui parler, quoi demander]
   Recommandation : [procéder / procéder avec prudence / passer]
```

### Dossier paysage concurrentiel

```
Cartographier le paysage concurrentiel pour [marché].

Définition marché : [décrire l'espace problème]
Position mon entreprise : [où nous jouons ou voulons jouer]
Objectif : [entrer marché / expansion / stratégie acquisition / pitch investisseur]

Structure paysage :

1. DÉFINITION MARCHÉ :
   Problème résolu : [en 2-3 phrases, du point de vue acheteur]
   Qui a ce problème : [profil acheteur — taille, industrie, rôle]
   Solutions actuelles : [comment acheteurs résolvent ceci aujourd'hui sans logiciel/service dédié]

2. TAILLE MARCHÉ :
   Estimation TAM : $[X] (source + méthodologie)
   SAM (serviceable) : $[X]
   Taux croissance : [X% CAGR — source]
   Stade : [émergeant / croissant / mature / déclinant]

3. MAPPAGE JOUEURS :

   Leaders catégorie (> $X ARR ou notable) :
   | Entreprise | Fondée | ARR/Financement | Cible | Différenciateur |
   |---|---|---|---|---|

   Challengers (croissants, notables) :
   [Même format]

   Spécialistes niche (fort dans segments spécifiques) :
   [Même format]

   Joueurs adjacents (peuvent expansion dans cet espace) :
   [Nom + pourquoi menace]

4. DYNAMIQUE CONCURRENTIELLE :
   Qui prend part marché et pourquoi ?
   Qui perd et pourquoi ?
   Catégorie se consolide ou se fragmente ?
   Quel est signal activité VC/investisseur ? (chaud = encombré et validé ; froid = opportunité ou mourant)

5. ANALYSE ESPACE BLANC :
   Segments mal desservis : [qui n'est pas bien servi ?]
   Lacunes fonctionnalités : [quoi acheteurs universellement se plaignent que personne ne fait bien ?]
   Lacunes géographiques : [marchés où aucun joueur fort existe]
   Lacunes go-to-marché : [canaux ou mouvements personne n'est bon]

6. OPPORTUNITÉ POSITIONNEMENT :
   Où pourrions-nous posséder position autres ne peuvent crédiblement prendre ?
   Quel 2x2 a le plus espace blanc ?
   Quel joueur, si nous les battons, devient notre récit principal ?
```

### Dossier technologie

```
Rechercher [technologie / framework / plateforme] pour décision construire-vs-acheter ou adoption.

Technologie : [nom]
Contexte décision : [évaluer pour adoption / comparer vs. alternatives / évaluer maturité]

Dossier technologie :

1. CE QU'C'EST :
   Objectif principal : [quel problème résout, en 2 phrases]
   Catégorie : [base de données / framework / plateforme / protocole / outil]
   Licence : [MIT / Apache / propriétaire / AGPL — implications affaires]
   Version actuelle et cadence release : [actif maintenu ?]

2. ADOPTION ET COMMUNAUTÉ :
   Étoiles GitHub : [X étoiles, X commits récents, X contributeurs]
   Téléchargements : [npm/PyPI/Docker Hub téléchargements hebdo]
   Taille communauté : [membres Discord/Slack, volume questions Stack Overflow]
   Adopteurs majeurs : [entreprises notables l'utilisant publiquement]
   Trajectoire : [croissant / stable / déclinant — vérifier historique étoiles]

3. ÉVALUATION TECHNIQUE :
   Forces principales : [ce qu'il fait mieux que alternatives]
   Limitations connues : [limites scaling, fonctionnalités manquantes, problèmes connus]
   Qualité documentation : [excellente / bonne / clairsemée / obsolète]
   Prêt production : [battle-testé / maturation / expérimental]
   Posture sécurité : [historique CVE, audit dépendances, politique divulgation sécurité]

4. ÉCOSYSTÈME :
   Intégrations : [quels outils et plateformes connectés nativement]
   Options gérées/hébergées : [si auto-hébergeable — offres gérées ?]
   Options support : [communauté-seul / support commercial / tier entreprise]
   Formation et ressources : [cours, tutoriels, livres]

5. COMPARAISON ALTERNATIVES :
   | Alternative | Forces | Faiblesses | Quand le choisir à la place |
   |---|---|---|---|

6. RECOMMANDATION :
   Adopter / Évaluer / Attendre / Éviter
   Raison : [1-2 raisons clés]
   Conditions adoption : [ce qui doit être vrai pour nous l'utilisons ?]
   Risque migration : [bas / moyen / élevé — si adoptons puis voulons changer]
```

## Exemple

**Utilisateur :** Nous évaluons l'acquisition d'une entreprise B2B SaaS à 3M ARR dans l'espace legal tech. J'ai besoin d'un dossier d'entreprise à présenter à notre conseil la semaine prochaine.

**Aperçu dossier de Claude pour cible acquisition legal tech :**

Donné contexte acquisition, voici ce qui importe au conseil — je structurerai le dossier pour répondre trois questions qu'un comité acquisition pose toujours :

**Question 1 : Ceci est une affaire qui vaut l'achat ?**
→ Section 3 (Modèle Affaires), Section 5 (Santé Financière), Section 6 (Signaux Client)

- Qualité revenu : ARR 3M est récurrent, ou mélange frais implémentation ponctuels ? NRR ? (> 100% = expansion ; < 85% = problème churn)
- Concentration client : Top 5 clients = quel % ARR ? (> 40% = risque concentration)
- Structure contrat : Annuels / multi-ans ? Terme contrat restant moyen pondéré ?

**Question 2 : Pouvons-nous l'intégrer sans le casser ?**
→ Section 4 (Produit), Section 2 (Leadership — risque personne clé)

- Y a-t-il fondateur technique qui partira à acquisition ? Si oui, risque transfert connaissances élevé.
- Comment le produit est construit ? (Pile tech de offres d'emploi et GitHub)
- Y a-t-il encumbrances IP ? (Licences OSS dans code base)

**Question 3 : Est-ce tarifé correctement ?**
→ Multiples transactions comparables pour legal SaaS à 3M ARR (typiquement 5-8x ARR pour haut NRR, 3-4x pour moyen)

Je trouverai cette information de :
- Crunchbase (financement, investisseurs, équipe fondation)
- G2 / Capterra (sentiment client, comparaisons concurrents)
- LinkedIn (taille équipe, modèles embauche, contextes leadership, départs)
- GitHub (si composants open source)
- Vérification dossiers court (PACER) pour litiges impliquant entreprise ou fondateurs
- Offres d'emploi (pile technologie, plans croissance, signaux financiers)

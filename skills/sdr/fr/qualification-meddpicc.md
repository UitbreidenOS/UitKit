# Qualification MEDDPICC

## Quand activer

- L'SDR a terminé l'appel de qualification initial et détermine s'il faut programmer une réunion de découverte avec un AE (ingénieur commercial / directeur de compte)
- Vous devez évaluer la qualité du deal sur les huit dimensions MEDDPICC avant la remise
- Évaluer s'il faut continuer à poursuivre un prospect lorsque les signaux de deal sont mitigés
- Construire un brief de remise pour l'AE avec des preuves documentées sur tous les éléments de qualification

## Quand NE PAS utiliser

- Utilisez ceci après le contact initial mais avant l'engagement AE — pas pour la post-découverte ou les comptes existants
- N'utilisez pas ceci pour les prospects entrants qui ont déjà passé le filtre initial — utilisez ceci pour décider *si* engager l'AE, pas pour valider l'implication de l'AE
- N'est pas un remplacement de l'écoute active ; utilisez ceci pour structurer les conclusions après la conversation, pas pendant
- Ne forcez pas les prospects marginaux à traverser le scoring MEDDPICC — si les éléments fondamentaux sont Inconnus/Manquants, cela signale le besoin d'une plus grande découverte, pas la préparation pour la remise

## Instructions

### Définition du cadre MEDDPICC

**M — Métriques** : Impact commercial quantifié. Demandez : « Si vous résolvez ce problème, que cela signifie-t-il en termes de revenu, d'économies de coûts, de temps économisé ou de réduction des risques ? Quel est l'impact financier ou opérationnel ? »
- Fort : Le prospect a articulé des chiffres spécifiques (X$ d'économies, Y% de gain d'efficacité, impact de Z revenu)
- Partiel : Le prospect a indiqué un impact mais non quantifié (« économiser de l'argent », « plus rapide », « réduire les erreurs »)
- Inconnu : Vous avez demandé mais le prospect ne pouvait pas quantifier
- Manquant : Pas encore de conversation sur l'impact

**E — Acheteur économique** : Qui contrôle le budget ? Avez-vous parlé directement à cette personne ou seulement à un influenceur ?
- Fort : Vous avez confirmé une conversation avec la personne qui a l'autorité budgétaire
- Partiel : Le prospect a confirmé qui est l'Acheteur économique mais vous ne lui avez pas parlé directement ; l'autorité budgétaire est probable
- Inconnu : Flou sur qui contrôle le budget ; le prospect esquive
- Manquant : Pas de conversation sur l'appropriation du budget

**D — Critères de décision** (premier D) : Quels sont les 3-5 critères d'évaluation principaux que le prospect utilisera pour juger les fournisseurs ?
- Fort : Le prospect a explicitement nommé 3+ critères par ordre de priorité (par ex., « 1. S'intègre à Salesforce, 2. Moins de 50k$/année, 3. Support 24h/24 »)
- Partiel : Le prospect a mentionné 1-2 critères mais pas exhaustif ou priorisé
- Inconnu : Vous avez demandé mais le prospect était vague (« nous voulons la meilleure solution »)
- Manquant : Pas de conversation sur comment ils évalueront

**D — Processus de décision** (deuxième D) : Qui est dans le comité d'achat ? Quelles sont les étapes ? Quel est le calendrier ?
- Fort : Le prospect a établi le comité (fondateur, directeur financier, responsable des opérations), le processus (RFP → démo → examen de sécurité → approbation) et le calendrier (4-6 semaines)
- Partiel : Le prospect a identifié certains acteurs et un calendrier approximatif mais le processus est flou
- Inconnu : Le prospect est flou sur les étapes ou les personnes impliquées
- Manquant : Pas de conversation sur le processus d'achat

**P — Processus de papier** : Approvisionnement, légal, examen de sécurité, approbations de conformité — qui possède chaque étape et combien de temps ?
- Fort : Le prospect a identifié chaque gate (par ex., « l'examen de sécurité prend 2 semaines, l'approvisionnement est 1 semaine, le légal est rapide »)
- Partiel : Le prospect a reconnu que certaines gates existent mais pas clair sur les propriétaires ou les calendriers
- Inconnu : Le prospect est incertain sur la rigueur de l'approvisionnement / du légal / de la sécurité
- Manquant : Pas de conversation sur les gates de signature

**I — Identifier la douleur** : Quelle est la douleur spécifique et quantifiée ? Que se passe-t-il s'ils ne font rien ?
- Fort : Le prospect a articulé une douleur spécifique et quantifiée (« Nous perdons 8 heures/personne/semaine sur la saisie manuelle de données ») et la conséquence de l'inaction (« Nous raterons les prévisions Q3 »)
- Partiel : Le prospect a mentionné une douleur mais pas spécifique ou quantifiée (« C'est un problème »)
- Inconnu : Le prospect a reconnu un problème mais ne le quantifiera pas ou n'articulera pas la conséquence
- Manquant : Pas de discussion sur la douleur ou la conséquence

**C — Champion** (premier C) : Avez-vous quelqu'un en interne qui vend pour vous quand vous n'êtes pas dans la salle ?
- Fort : Personne identifiée qui évangélisera, a déclaré l'engagement, a des intérêts en jeu (par ex., son flux de travail s'améliore)
- Partiel : Champion probable identifié mais engagement flou ou ils ont des incitatifs mixtes
- Inconnu : Le prospect a suggéré un champion possible mais vous ne l'avez pas validé
- Manquant : Pas de champion identifié ; tout le monde est neutre ou sceptique

**C — Concurrence** (deuxième C) : Quels fournisseurs évaluent-ils ? Qu'aiment-ils/détestent-ils à propos de ces options ?
- Fort : Le prospect a nommé 2+ concurrents, a déclaré ce qui est bon/mauvais à propos de chacun par rapport à ce dont ils ont besoin
- Partiel : Le prospect a mentionné 1 concurrent ou des fonctionnalités qu'il aime/déteste mais pas de comparaison concurrentielle
- Inconnu : Le prospect a esquivé ou dit « nous n'avons pas examiné les alternatives »
- Manquant : Pas de conversation sur l'ensemble concurrentiel

---

### Logique de scoring MEDDPICC

Notez chaque élément sur une échelle 0-3 :
- **3 = Fort** : Toutes ou presque toutes les informations présentes, quantifiées et confirmées directement par le prospect
- **2 = Partiel** : Certaines informations présentes ; peut manquer de quantification, de confirmation par le décideur ou de complétude
- **1 = Inconnu** : Vous avez demandé mais avez reçu une réponse vague ou évasive
- **0 = Manquant** : Pas de conversation sur cet élément ou le prospect a explicitement dit « nous ne savons pas encore »

**Score SAP (Sales Accuracy Predictor)** : Additionnez les huit éléments. Maximum = 24.

Interprétation :
- **20-24 (Excellent)** : 6-8 éléments à Fort ; prêt pour la remise AE ; probabilité de fermeture élevée
- **14-19 (Bon)** : Mélange de Fort et Partiel ; répond au seuil de remise ; l'AE devrait confirmer Pain/Champion/Métriques en découverte
- **12-13 (Prudence au seuil)** : Plus Partiel que Fort ; remise marginale ; escalader au manager si Économique/Processus de décision sont faibles
- **0-11 (Pas prêt)** : Trop d'Inconnu/Manquant ; nécessite un autre appel de découverte avant l'engagement AE ; programmer un suivi

**Seuil de remise** : L'SDR doit atteindre **un score SAP de 14+** avant de programmer la découverte AE. Ne programmez pas l'AE pour un score < 14 sans approbation du manager.

### Questions de découverte par élément MEDDPICC

Utilisez ces questions pour sonder chaque dimension lors de l'appel de qualification :

**Métriques :**
- « Si vous implémentez une solution à ce problème, à quoi ressemble le succès en termes financiers ou opérationnels ? Quel est l'impact en dollars, heures économisées ou réduction des risques ? »
- « Comment mesurez-vous le coût du problème aujourd'hui ? Quel est le fardeau annuel ? »
- « Quels KPI changeraient une fois que ce sera résolu ? »

**Acheteur économique :**
- « Qui possède le budget pour une solution comme celle-ci ? »
- « Êtes-vous la bonne personne pour parler du budget, ou devrions-nous inclure [leur nom] à notre prochain appel ? »
- « Qui d'autre devrait approuver les dépenses dans votre organisation pour quelque chose comme cela ? »

**Critères de décision :**
- « Lorsque vous évaluez les fournisseurs, quels sont les 3-5 points qui comptent le plus pour vous ? Dans quel ordre ? »
- « Qu'est-ce qui est non-négociable par rapport à ce qui est agréable à avoir ? »
- « Si vous deviez choisir entre [prix/intégration/support], lequel gagne ? »

**Processus de décision :**
- « Décrivez-moi comment une décision d'achat se déroulerait dans votre organisation. Qui doit approuver ? »
- « Y a-t-il des étapes formelles comme un RFP, une démo, un examen de sécurité ou un examen juridique ? »
- « Quel est le calendrier typique de la première conversation à la signature du contrat ? »

**Processus de papier :**
- « Avez-vous un processus d'approvisionnement ? Qui le possède et combien de temps cela prend-il généralement ? »
- « Y a-t-il un examen de sécurité ou une approbation de conformité ? Qui s'en charge ? »
- « Le légal examine-t-il les contrats des fournisseurs ? Combien de temps cela prend-il ? »

**Identifier la douleur :**
- « Quelle est la douleur spécifique que vous ressentez aujourd'hui ? Pouvez-vous la décrire en termes de ce qui est cassé ou vous ralentit ? »
- « À quelle fréquence ce problème se pose-t-il ? Combien de temps faut-il pour y contourner ? »
- « Que se passe-t-il si vous ne résolvez pas cela d'ici [date cible] ? Quel est le coût de ne rien faire ? »

**Champion :**
- « Qui sur votre équipe bénéficiera le plus de résoudre ce problème ? »
- « Y a-t-il quelqu'un dans votre organisation qui a plaidé pour une solution à ce problème ? »
- « Qui vous intégreriez-vous naturellement lorsque vous discutez d'un fournisseur potentiel ? »

**Concurrence :**
- « Évaluez-vous d'autres solutions ou fournisseurs pour cela ? »
- « Qu'est-ce qu'ils regardent ? Ont-ils fait des démos ou des pilotes ? »
- « Qu'aimez-vous ou détestez-vous à propos des options que vous envisagez ? »

### Pièges courants et comment les éviter

**Élément le plus communément manqué : Concurrence**

La plupart des SDR ignorent la découverte concurrentielle parce que :
- Ils craignent que cela fasse penser au prospect « il y a d'autres options » (en réalité, la plupart des entreprises ÉVALUENT plusieurs fournisseurs)
- Ils ne savent pas comment demander sans paraître défensifs
- Ils supposent que le prospect volontairement partagera les informations concurrentielles (ils ne le feront pas)

**Solution :**
- Demandez tôt, encadrez comme « comprendre le paysage » : « Travaillez-vous actuellement avec d'autres fournisseurs sur cela ? »
- Reformulez comme validation du processus : « La plupart des équipes avec lesquelles je parle évaluent 2-3 options. Qui d'autre regardez-vous ? »
- Utilisez la réponse du prospect pour différencier : « Ils sont forts sur [fonctionnalité], mais luttent avec [fonctionnalité qu'on fait bien]. Est-ce une préoccupation ? »

**Autres lacunes courantes :**

- **Métriques inconnues** : Le prospect dit « économiser du temps » mais ne quantifie pas. Action : « Si vous économisez 1 heure/personne/semaine, qu'est-ce que cela vaut en coût salarial ? » (poussez-les, n'acceptez pas vague)
- **Acheteur économique manquant** : Vous n'avez parlé qu'à un influenceur. Action : Programmer un appel avec le propriétaire du budget *avant* la remise AE. Ne demandez pas à l'influenceur de faire la présentation et espérez que cela colle.
- **Processus de décision vague** : Le prospect dit « nous allons simplement le soumettre à la direction ». Action : « Qui spécifiquement est dans votre équipe de direction ? Qui décide des achats de logiciels ? »
- **Processus de papier inconnu** : Le prospect dit « nous allons nous en charger ». Action : « Y aura-t-il une équipe d'approvisionnement, un examen de sécurité ou un examen juridique ? Qui possède ceux-ci ? »
- **Champion faible** : Le prospect est neutre ou vous n'avez pas identifié de champion. Action : « Qui sur votre équipe ressentirait le plus d'impact si nous résolvions cela ? » (trouvez la personne avec des intérêts en jeu)

---

### Règles de scoring

- **N'gonfllez pas les scores pour la rapport** : Si un prospect est sympathique mais a donné des réponses vagues, score = 1 (Inconnu), pas 2 (Partiel)
- **Ne notez pas par inférence** : Si le prospect n'a pas explicitement dit que le budget existe, ne supposez pas que l'Acheteur économique est Fort
- **Partiel ≠ Bon** : Partiel signifie que vous avez certaines données mais des lacunes demeurent. Si deux éléments sont Partiels, c'est un risque matériel dans la progression du deal
- **Inconnu ≠ Partiel** : Inconnu = vous avez demandé et avez obtenu une réponse évasive. Partiel = vous avez obtenu certaines données mais pas toutes. Ce sont des signaux différents
- **Si SAP < 14, n'escaladez pas à l'AE** : Programmer un deuxième appel de découverte pour fermer les lacunes critiques (E, I, C Champion)

---

### Arbre de décision : Remettre ou pas ?

```
Notez les 8 éléments (0-3 chacun)

SAP ≥ 20?
├─ OUI → Remettez immédiatement. L'AE confirme en découverte.
├─ NON, SAP 14-19? → Remettez. Préparez l'AE avec des lacunes. Marquez les éléments les plus faibles.
└─ NON, SAP 12-13?
   ├─ E (Acheteur économique) ou I (Identifier la douleur) à 0 ou 1? → NE PAS remettre. Programmer un suivi.
   ├─ Processus de décision ou processus de papier à 0 ou 1? → Remettez seulement si les autres 5 sont 2+. Marquez le risque.
   └─ Autrement → Examen du manager. Si vous avez établi la rapport, ça vaut peut-être un appel de plus.

SAP < 12?
└─ NE PAS remettre. Programmer une découverte de suivi. Le prospect n'est pas prêt.
```

---

### Traduction des termes MEDDPICC pour les plus grands comités d'achat

Lorsque le prospect dit « nous n'avons pas un seul acheteur », mappez à leur structure :
- **Acheteur économique** = « Propriétaire du budget » (CFO, VP Finance, Responsable de l'approvisionnement)
- **Champion** = « Parrain interne » ou « utilisateur puissant » (personne qui évangélisera)
- **Critères de décision** = « Cadre d'évaluation » ou « rubrique de notation »
- **Processus de papier** = « Liste de contrôle de signature » ou « gates d'approbation »

## Exemple

**Scénario : Outil SaaS de renseignement commercial. Vous (SDR) en appel avec Maya, directrice du marketing chez une fintech en phase Series-B.**

**Transcription de l'appel (extraits) :**

*Vous : « Maya, si nous résolvons le défi d'obtenir un scoring des leads précis, que cela signifie-t-il pour votre équipe ? »*

*Maya : « Honnêtement, nous perdons peut-être 20-30% des leads qualifiés à cause de mauvaises données. Si nous réglions cela, nous ajouterions probablement 500k$ en revenu annuel juste en améliorant notre taux de fermeture. »*

→ **Métriques : 3 (Fort)** — Impact spécifique quantifié (500k$ revenu)

---

*Vous : « Qui devrait approuver un outil comme celui-ci ? »*

*Maya : « Ce serait notre CFO, James. Il possède les dépenses logicielles. J'ai mentionné le problème à ce sujet mais n'ai pas formellement demandé un budget. »*

→ **Acheteur économique : 2 (Partiel)** — Identifié mais non directement confirmé ; statut du budget flou

---

*Vous : « Lorsque vous évaluez une solution, qu'est-ce qui compte le plus ? »*

*Maya : « Nous avons besoin qu'elle s'intègre à Salesforce — c'est non-négociable. Deuxièmement, le prix compte ; nous sommes bootstrap donc moins de 50k$/année. Troisièmement, la vitesse d'intégration ; nous devons avancer vite. Et honnêtement, la réputation du fournisseur en fintech aide. »*

→ **Critères de décision : 3 (Fort)** — Quatre critères clairs avec classement par priorité

---

*Vous : « Décrivez-moi comment cela serait approuvé. Y a-t-il d'autres étapes au-delà de James ? »*

*Maya : « Oui, donc James approuverait les dépenses. Notre VP des ventes devrait approuver la solution elle-même. Et parce que nous gérons les données financières, il y a un examen de sécurité — cela prend généralement 2 semaines. Ensuite, le légal ferait un examen rapide du contrat, peut-être 1 semaine. Calendrier ? Probablement 4-5 semaines de la décision au contrat. »*

→ **Processus de décision : 3 (Fort)** — Parties prenantes, calendrier et structure clairs
→ **Processus de papier : 3 (Fort)** — Examen de sécurité (2 semaines), légal (1 semaine), propriétaires d'approvisionnement identifiés

---

*Vous : « Qu'est-ce qui crée l'urgence autour de cela ? Que se passe-t-il si vous ne résolvez pas le problème du scoring des leads ? »*

*Maya : « Nous augmentons les ventes en Q3. Si nous ne réglons pas la qualité des leads maintenant, nous raterons nos objectifs de pipeline de probablement 25-30%. James sait que c'est un risque. »*

→ **Identifier la douleur : 3 (Fort)** — Douleur spécifique et quantifiée (25-30% manquement du pipeline) et conséquence (manque Q3)

---

*Vous : « Qui sur votre équipe serait le plus enthousiaste à propos de cette solution ? »*

*Maya : « Définitivement notre Responsable des ventes, David. Il a plaidé pour de meilleures données depuis un an. Si je lui montre une solution viable, il se battra pour cela en interne. »*

→ **Champion : 3 (Fort)** — Identifié, motivé et positionné pour évangéliser

---

*Vous : « Évaluez-vous d'autres fournisseurs pour cela ? »*

*Maya : « Nous avons regardé Clearbit l'année dernière, mais leurs tarifs ont augmenté. Nous sommes aussi en discussions initiales avec [concurrent X], qui est moins cher mais ne s'intègre pas aussi proprement à Salesforce. Votre histoire d'intégration est meilleure. »*

→ **Concurrence : 3 (Fort)** — Concurrents nommés, pro/cons énoncés

---

**SCORECARD SAP :**
```
M — Métriques :            3 (500k$ impact revenu)
E — Acheteur économique :  2 (Identifié mais non confirmé)
D — Critères de décision : 3 (4 critères, priorisés)
D — Processus de décision :3 (Parties prenantes + calendrier)
P — Processus de papier :  3 (Sécurité + légal + calendrier)
I — Identifier la douleur :3 (25-30% manquement pipeline, conséquence Q3)
C — Champion :             3 (David, Responsable des ventes, motivé)
C — Concurrence :          3 (Clearbit, Concurrent X, comparaison énoncée)

SCORE SAP TOTAL : 23/24
```

**Décision** : Remettez immédiatement à l'AE. Ce deal est prêt pour l'entreprise.

**BRIEF DE REMISE à l'AE :**
```
Prospect : Maya Lopez, Directrice du marketing, Fintech en phase Series-B
Acheteur économique : James (CFO) — PAS ENCORE PARLÉ À
Champion : David (Responsable des ventes) — Motivé, évangélisera

Score SAP : 23/24

Métriques clés : 500k$ impact revenu de la correction de la qualité des leads ; 25-30% manquement du pipeline si non résolu d'ici Q3

Lacunes à confirmer en découverte :
- Appétit de James pour les dépenses (Maya n'a pas formellement demandé encore)
- Calendrier de David pour la solution (doit s'aligner sur la montée Q3)

Suivant : Programmer la démo avec Maya + David ; intégrer James pour la discussion budgétaire.

Drapeaux de risque : Aucun. Deal fort.
```

---

Ce prospect franchit le seuil de remise avec un SAP de 23/24. Le travail de l'AE est de verrouiller l'approbation de l'Acheteur économique et de confirmer le calendrier.

# Multi-Agent Memory Vaardigheid

## Wanneer activeren

- Multi-agent systemen bouwen waar agenten gedeelde state moeten openen/wijzigen
- Blackboard-patroon architecturen implementeren
- Memory conflicten behandelen als meerdere agenten dezelfde data gelijktijdig updaten
- Agent geheugen ontwerpen dat over meerdere workflow executes persisteert
- Agent team geheugen implementeren (episodisch en semantisch kennis)

## Wanneer NIET gebruiken

- Single-agent workflows met privé geheugen
- Read-only workflows zonder gedeelde mutable state
- Workflows waar agenten nooit elkaars data openen

# Compétence Directives de Marque

## Quand l'activer
- Créer les directives de marque pour une nouvelle entreprise ou un nouveau produit
- Documenter votre marque pour les nouveaux membres de l'équipe, contractants ou agences
- Auditer la cohérence de marque entre les canaux
- Définir les règles pour le contenu généré par l'IA pour rester on-brand
- Mise à jour des directives de marque après un rebrand

## Quand ne pas l'utiliser
- Exécution de conception visuelle (création de logo, palettes de couleurs) — nécessite un designer
- Stratégie de marque à partir de zéro sans direction existante — nécessite un travail de découverte
- Travail de marque légale — consulter un avocat PI

## Instructions

### Voix et ton de marque

```
Définir la voix et le ton pour [entreprise/produit].

Contexte de l'entreprise :
- Ce que nous faisons : [décrivez]
- Qui nous servons : [audience cible]
- Notre personnalité : [3-5 adjectifs — p.ex. "direct, chaleureux, expert, sans détour"]
- Ce que nous ne sommes PAS : [3-5 opposés — p.ex. "pas corporatiste, pas jargonneux"]

Définir :
1. Voix (cohérente toujours) : [3-4 attributs de voix fondamentale avec descriptions]
2. Ton (change par contexte) : comment le ton change pour :
   - Annonces de produits
   - Support client / messages d'erreur
   - Copy marketing
   - Documentation technique
   - Médias sociaux
   - Communications de crise

Inclure : 2-3 avant/après exemples pour chaque contexte de ton.
```

### Hiérarchie de messages

```
Construire la hiérarchie de messages pour [entreprise/produit].

Information de base :
- Quel problème résolvons-nous ? [décrire]
- Qui est le client principal ? [ICP]
- Comment nous le résolvons différemment que les alternatives ? [différenciation]
- Quel résultat les clients obtiennent-ils ? [avantage final]

Niveaux de messages :
1. LIGNE UNIQUE : 8-12 mots. Utilisé dans les bios, publicités, conversation.
2. PITCH D'ASCENSEUR : 2-3 phrases. Utilisé dans appels d'intro, héro site web, intros email.
3. PARAGRAPHE : 5-6 phrases. Utilisé dans pages À propos, propositions, kits presse.
4. RÉCIT COMPLET : L'histoire de marque plus longue (quand, pourquoi, qui, et enjeu).

Aussi définir :
- Proposition de valeur (ce que nous offrons)
- Mission (pourquoi nous existons)
- Vision (monde vers lequel nous nous dirigeons)
- Options de slogan (3 variantes pour test)
```

### Règles de style d'écriture

```
Définir les règles de style d'écriture pour [entreprise].

Créer un guide de style pratique couvrant :

VOCABULAIRE
- Mots que nous possédons (utiliser systématiquement) : [p.ex. "construire" pas "créer", "envoyer" pas "lancer"]
- Mots que nous évitons : [jargon, buzzwords, phrases surutilisées]
- Règles de capitalisation : [caps du nom de produit, caps du nom de feature]
- Nombres : [épeler sous 10 / utiliser des chiffres]

FORMATAGE
- Longueur de phrase : [court et percutant / plus long et réfléchi]
- Longueur de paragraphe : [max X lignes par paragraphe]
- Virgule Oxford : [oui / non]
- Contractions : [les utiliser / éviter dans contextes formels]
- Tirets et ellipses : [préférence de style]

PRINCIPES DE CONTENU
- Voix active vs passive : [toujours actif / exceptions]
- Comment nous parlons des concurrents : [règles]
- Comment nous parlons des clients : [utilisons-nous "utilisateurs" ou "clients" ou "gens" ?]
- Comment décrire la tarification : [règles]

Résultat : une carte de référence rapide d'une page que les rédacteurs peuvent utiliser.
```

### À faire et à ne pas faire de marque

```
Créer une liste à faire et à ne pas faire pour [canal/cas d'utilisation].

Contexte : [médias sociaux / publicité / communiqués / support / onboarding]

Pour chaque règle, inclure :
- La règle (claire, actionnable)
- Pourquoi cela importe (1 phrase)
- Exemple de bien faire
- Exemple de mal faire

Couvrir :
1. Ton et langage (5 règles)
2. Identité visuelle (5 règles) — même si juste décrire l'intention
3. Affirmations et preuves (3 règles) — ce que nous pouvons et ne pouvons pas dire
4. Mentions des concurrents (2 règles)
5. Histoires de clients (2 règles — consentement, exactitude)
```

### Directives de contenu généré par l'IA

```
Écrire les directives de contenu généré par l'IA.

À mesure que nous utilisons Claude/ChatGPT pour créer du contenu, quelles règles doivent être suivies ?

Définir :
1. Quoi l'IA PEUT écrire sans review : [p.ex. légendes sociales, lignes d'objet email, premiers brouillons]
2. Quoi l'IA NE DOIT PAS écrire sans review humain : [p.ex. affirmations, langage légal, comms de crise]
3. Comment vérifier la sortie IA pour la voix de marque avant publication : [checklist 3-étapes]
4. Phrases-drapeaux que l'IA tend à utiliser qui ne correspondent pas à notre marque : [liste basée sur notre voix]
5. Comment promper l'IA pour écrire dans notre voix : [le system prompt ou instructions clés]
```

## Exemple

**Utilisateur :** Écrire les directives de voix de marque pour une startup d'outils pour développeurs. Nous voulons être vu comme des pairs experts, pas des spécialistes du marketing.

**Résultat de Claude :**

**Voix : Pair Expert**

| Attribut | Ça signifie | Exemple |
|---|---|---|
| Direct | Dites ce que vous voulez dire. Pas de rinçage de gorge. | "Cela coupe votre temps CI de moitié." pas "Nous sommes enthousiastes de partager..." |
| Technique | Ne simplifiez pas. Faites confiance au lecteur. | Utilisez les bons termes techniques ; n'expliquez pas les bases |
| Humain | Nous sommes aussi des ingénieurs. | "Nous nous sommes frustrés avec X, donc nous avons construit Y." |
| Précis | Affirmations spécifiques, pas promesses vagues | "P50 sous 50ms" pas "ridiculement rapide" |

**Ce que nous ne disons jamais :** leverage, synergy, game-changing, seamless, best-in-class, innovative, cutting-edge, world-class, excited to announce, thrilled to share.

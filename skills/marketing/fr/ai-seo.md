# Compétence SEO IA

## Quand l'activer
- Optimiser le contenu pour apparaître dans les aperçus IA de Google (anciennement SGE)
- Être cité par ChatGPT, Perplexity ou Claude quand les utilisateurs posent des questions
- Adapter votre stratégie SEO à mesure que la recherche IA prend des parts de la recherche traditionnelle
- Écrire du contenu structuré pour l'extraction et la citation par l'IA
- Auditer le contenu existant pour la compatibilité avec la recherche IA

## Quand ne pas l'utiliser
- SEO traditionnel des liens bleus (classement #1 pour les termes de tête) — utiliser la compétence seo-audit
- Recherche payante sur les plates-formes IA — encore en évolution, pas de meilleure pratique établie
- Sites entièrement nouveaux sans autorité — d'abord construire les fondamentaux

## Instructions

### Comprendre l'intention de recherche IA

```
Analyser cette requête pour l'optimisation de recherche IA :

Requête : "[collez la requête]"
Ma page/contenu : [décrivez ou URL]

Dites-moi :
1. Est-ce une requête navigationnelle, informationnelle ou transactionnelle ?
2. Google afficherait-il un aperçu IA pour cette requête ? (probable pour informationnelle, how-to, comparaisons)
3. Quel format prend généralement l'extrait IA ? (liste / paragraphe / tableau / étapes)
4. Que doit contenir mon contenu pour être cité ?
5. Quelle est la structure de contenu idéale pour cette requête ?
```

### Optimiser le contenu pour les aperçus IA

```
Optimiser ce contenu pour les aperçus IA de Google.

Contenu : [collez ou décrivez]
Requête cible : "[collez]"

Structurez le contenu pour que :
1. La réponse TLDR apparaisse dans les 40-50 premiers mots
2. Les points clés soient dans un format scannable (H2/H3 + paragraphes courts)
3. Les faits, statistiques et définitions soient clairement énoncés
4. Le contenu réponde directement à la question avant d'élaborer
5. Les questions connexes soient couvertes (motifs People Also Ask)

Réécrire l'introduction et les sections clés pour prioriser les réponses directes.
```

### AEO — Optimisation des moteurs de réponse

```
Écrire du contenu optimisé pour être cité par Perplexity, ChatGPT et Claude.

Sujet : [décrivez]
Question cible : "Comment je [X] ?" ou "Qu'est-ce que [X] ?"

Ces moteurs IA préfèrent :
- Des déclarations claires et factuelles avec nombres et données spécifiques
- Information structurée (définition → pourquoi c'est important → comment → exemple)
- Ton d'autorité sans hésitation ("X est" pas "X pourrait être")
- Perspective unique ou données (la recherche originale bat le contenu agrégé)
- Contenu à jour (les moteurs IA pénalisent les informations obsolètes)

Écrire du contenu qu'une IA citerait quand un utilisateur demande à propos de [sujet].
```

### Signaux E-E-A-T pour l'ère IA

```
Auditer cette page pour les signaux E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness) :

Page : [URL ou description]

Les moteurs de recherche IA pondèrent E-E-A-T lourdement. Vérifier :
- Bio de l'auteur avec identifiants visibles ?
- Signaux d'expérience de première main ("J'ai testé ceci pendant 6 mois...")
- Données originales, recherche ou études de cas ?
- Citations externes et sources liées ?
- Date clairement affichée et tenue à jour ?
- L'auteur a une présence externe (LinkedIn, publications, présentations) ?

Pour chaque écart : action spécifique pour ajouter des signaux E-E-A-T.
```

### Schéma FAQ pour les citations IA

```
Écrire un balisage de schéma FAQ pour cette page.

Sujet : [décrivez]
URL de la page : [URL]
Questions communes que les utilisateurs posent : [listez ou laissez-moi les générer]

Générer :
1. 5-8 paires FAQ (question + réponse concise)
2. Balisage de schéma JSON-LD prêt à coller dans <head>
3. Section FAQ en ligne à ajouter à la page (pas seulement du schéma)

Chaque réponse doit être de 40-60 mots — assez de contexte pour que l'IA cite, assez court pour afficher en extraits.
```

### Monitorer la visibilité de recherche IA

```
Créer un plan de monitoring pour la visibilité de recherche IA.

Mon site : [décrivez]
Sujets clés pour lesquels j'ai un classement : [liste]

Suivre :
1. Apparitions dans les aperçus IA de Google (Search Console : search_type=discover + rapport des features IA)
2. Citations Perplexity (rechercher "[site:monsite.com]" mensuellement + vérifications manuelles)
3. Visibilité ChatGPT (demander "Quels sont les meilleurs outils pour X ?" mensuellement, vérifier si cité)
4. Taux zero-click dans GSC (impressions montantes, clics plats = l'IA répond à vos requêtes)

Déclencheurs d'alerte : [ce qui devrait déclencher les mises à jour de contenu]
Cadence de review : [mensuelle / trimestrielle]
```

## Exemple

**Utilisateur :** Mon article de blog "meilleurs outils de gestion de projets" reçoit 5 000 recherches/mois mais les clics ont baissé de 40% depuis que Google a ajouté les aperçus IA.

**Solution SEO IA de Claude :**

1. Restructurer l'intro : mener avec un tableau de comparaison direct (les aperçus IA adorent les tableaux pour les requêtes "meilleur X")
2. Ajouter une boîte "Réponse rapide" au sommet : "Le meilleur outil de gestion de projets pour les petites équipes est X parce que [raison]. Pour entreprise : Y."
3. Ajouter une section FAQ avec 6 questions spécifiques que l'IA afficherait : "Notion est-il mieux que Asana ?", "Quelle est la meilleure option gratuite ?", etc.
4. Ajouter le balisage de schéma FAQ JSON-LD
5. Inclure des données originales : "Nous avons testé 8 outils pendant 3 mois — voici ce que nous avons trouvé" signale E-E-A-T
6. Accepter la réalité zero-click : optimiser pour la notoriété de marque et les clics en bas de l'entonnoir, pas le trafic au milieu de l'entonnoir

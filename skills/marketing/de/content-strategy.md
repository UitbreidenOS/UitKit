# Content Strategy Skill

## Wann aktivieren
- Aufbau einer Content-Strategie von Grund auf für eine Marke oder ein Produkt
- Gestaltung von Inhaltssäulen und Themenclustern
- Erstellung eines Redaktionskalenders über Kanäle hinweg
- Aufbau eines Content-Umgestaltungssystems (eine Komponente → viele Formate)
- Ausrichtung des Inhalts auf SEO-Ziele und Geschäftsziele
- Überprüfung vorhandener Inhalte auf Lücken und Möglichkeiten

## Wann NICHT verwenden
- Schreiben einer einzelnen Komponente — schreibe es einfach, strategisiere nicht
- Technische SEO-Audits — verwende den seo-audit Skill
- Bezahlte Content-Verteilung — verwende den paid-ads Skill
- Nur Social-Media-Posting-Kadenz — verwende den social-media-manager Skill

## Anweisungen

### Definieren Sie Inhaltssäulen

```
Definieren Sie Inhaltssäulen für [Marke/Produkt].

Geschäft: [beschreiben Sie, was Sie tun und wen Sie bedienen]
Zielgruppe: [Jobtitel, Unternehmensgröße, Schlüsselherausforderungen]
Geschäftsziele, die der Inhalt unterstützen sollte: [Markenbewusstsein / Lead-Generierung / Kundenbindung / SEO-Traffic]
Konkurrenten, die Inhalte erstellen: [wer und worüber sie schreiben]
Einzigartiger Winkel, den wir haben: [Expertise, Standpunkt oder Daten, die wir besitzen]

Entwerfen Sie 3-5 Inhaltssäulen:
Jede Säule = ein Thementerritorium, das wir besitzen und regelmäßig veröffentlichen.

Format pro Säule:
- Säulenname: [klare Bezeichnung]
- Warum diese Audience es interessiert: [der zugrunde liegende Schmerz oder das Ziel]
- Inhaltstypen, die passen: [how-to / Meinung / Fallstudie / Daten / Nachrichten-Kommentar]
- Unser einzigartiger Winkel: [warum wir darüber besser sprechen können als jeder andere]
- Beispiel-Überschrift: [spezifische Artikel-/Posttitel]

Regel: Säulen sollten spezifisch genug sein, um besitzergreifend zu sein, breit genug für 52 Stücke/Jahr.
```

### Redaktionskalender

```
Bauen Sie einen 90-Tage-Redaktionskalender für [Marke].

Zu abdeckende Kanäle: [Blog / Newsletter / LinkedIn / YouTube / Twitter/X]
Veröffentlichungshäufigkeit: [X Posts/Woche pro Kanal]
Inhaltssäulen: [oben einfügen]
Bevorstehende Kampagnen oder Starts: [Produktstarts, Veranstaltungen, Jahreszeiten]
Ressourcen: [1 Schriftsteller / 2 Schriftsteller / Team]

Kalenderformat:
Woche | Kanal | Säule | Titel | Format | CTA | Notizen

Regeln:
- Beginnen Sie mit SEO-gesteuerten Artikeln (Blog) — höchster langfristiger ROI
- Newsletter: Kuratieren + Perspektive hinzufügen, nicht einfach Blog-Posts erneut veröffentlichen
- LinkedIn: Kurzversionen von Blog-Insights (3-5 Zeilen + Hook)
- Ähnliche Formate zusammen batchen für Produktionseffizienz
- Mindestens 2 Wochen im Voraus planen; 10% Slots für reaktive/Trending-Inhalte verlassen

Produktion: 12-Wochen-Kalender mit 1 Anker-Komponente pro Woche + Verteilungsplan für jede.
```

### Content-Umgestaltungssystem

```
Entwerfen Sie ein Content-Umgestaltungssystem.

Primärer Kanal und Format: [Blog / Podcast / YouTube / Live-Veranstaltung]
Sekundäre Kanäle zum Füttern: [alle auflisten: Newsletter, LinkedIn, Twitter, YouTube Shorts, etc.]
Teamgröße: [solo / 1 Redakteur / vollständiges Team]
Veröffentlichungstakt: [X primäre Komponenten pro Monat]

Umgestaltungskarte (1 primär → viele):

Wenn primär = Langformartikel (2.000+ Wörter):
→ Newsletter: Intro + 3 Schlüsselpunkte + Link
→ LinkedIn: der kontraintuitivste Einblick, einzelner Post
→ Twitter/X: Thread von 5-7 Punkten
→ Kurzvideo: 60-Sekunden-Zusammenfassung "Was ich gelernt habe"
→ Zitatkarte: 1 gezogenes Zitat → Bild für Instagram/LinkedIn

Wenn primär = Podcast-Episode:
→ Blogartikel: in Artikel bearbeitete Abschrift (Headers hinzufügen, Beispiele erweitern)
→ Newsletter: Episoden-Zusammenfassung + Top 3 Takeaways
→ LinkedIn: Gastzitat oder Schlüsseleinsicht als Post
→ YouTube Shorts: bestes 60-Sekunden-Clip
→ Twitter: Schlüsselstat oder Zitat-Thread

Bauen Sie das spezifische System für meine Kanäle mit Aufträgen pro Komponente.
```

### Content-Audit

```
Prüfen Sie vorhandene Inhalte und identifizieren Sie Möglichkeiten.

Inhaltsstandort: [Blog-URL / URL-Liste / Sitemap]
Geschäftsziele: [Testanmeldungen antreiben / für X Keywords ranken / Kunden halten]
Aktuelles Inhaltsvolumen: [X Artikel / X Posts]
Analytics-Zugriff: [GA4 / Search Console / keine]

Audit-Rahmen:
1. Traffic-Segmente:
   - Hoher Traffic, hohe Konversion → schützen und aktualisieren
   - Hoher Traffic, niedrige Konversion → CTA und Landungsfluss optimieren
   - Niedriger Traffic, hohe Konversion → mehr fördern (bezahlt, Links, E-Mail)
   - Niedriger Traffic, niedrige Konversion → aktualisieren oder löschen

2. SEO-Chancen-Scan:
   - Artikel Ranking-Positionen 8-20 → aktualisieren, um auf Seite 1 zu drücken
   - Artikel mit Null-Organik-Traffic nach 6 Monaten → konsolidieren oder umleiten
   - Fehlende Themen, für die Konkurrenten ranken → erstellen

3. Inhaltsfrische:
   - Artikel > 18 Monate alt → Faktencheck und Aktualisierung
   - Artikel, die auf veraltete Tools, Stats oder Praktiken verweisen → aktualisieren

Ausgang: priorisierte Liste von:
- [X] Artikel zum Aktualisieren (schnelle Erfolge, Ranking 8-20)
- [X] Artikel zum Konsolidieren (dünner Inhalt)
- [X] nettoneu zu erstellende Themen (Lückenanalyse)
- [X] Artikel zum Löschen + Umleiten
```

### Verteilungsplan

```
Bauen Sie einen Verteilungsplan für jede neue Komponente.

Inhaltstyp: [Artikel / Video / Podcast-Episode / Bericht]
Zielgruppengröße nach Kanal: [E-Mail: X / LinkedIn: X Follower / Twitter: X / etc.]
Promotionsbudget: $[X] (oder $0 für nur organisch)

Verteilungssequenz (Tag für Tag):

Tag 0 — Veröffentlichen:
□ Auf dem primären Kanal veröffentlichen (Titel, Meta, OG-Bild optimieren)
□ An E-Mail-Liste senden (vollständiger Post oder Ausschnitt + Link)
□ Auf LinkedIn posten (angepasste Version, nicht nur Link)
□ Auf Twitter/X posten (Thread oder einzelnes Zitat)

Tag 3 — Verstärkung:
□ Engagieren Sie sich mit allen Kommentaren auf allen Kanälen
□ LinkedIn-Kommentar auf Ihrem eigenen Post mit Follow-up-Einsicht veröffentlichen
□ In relevanten Gemeinschaften teilen (Slack, Discord, Reddit — Wert hinzufügen, keinen Spam)

Tag 7 — Umgestaltung:
□ YouTube Short oder Reel veröffentlichen (falls Videoformat verfügbar)
□ Zitat als Bild-Post umgestalten

Tag 30 — SEO-Follow-up:
□ Rankings überprüfen — erscheint es für das Ziel-Keyword?
□ Interne Links von neuerer Inhalte zu diesem Stück hinzufügen

Bezahlte Verstärkung (falls Budget):
- Beste LinkedIn-Posts boosten ($50-100 pro Post)
- Google Discovery oder Meta Retargeting für Blog-Leser (Bewusstsein)
```

## Beispiel

**Benutzer:** Ich führe ein B2B SaaS Unternehmen (Projektmanagement-Tool für Engineering-Teams). Keine Content-Strategie, wir schreiben einfach zufällig Blog-Posts. Helfen Sie mir, eine aufzubauen.

**Claudes Content-Strategie:**

**3 Inhaltssäulen:**

1. **Engineering Team Produktivität** — "Wie Engineering Teams schneller verschieben, ohne Burnout"
   - Audience-Schmerz: Versand-Geschwindigkeit ohne Qualitätsopfer
   - Formate: How-to-Guides, Benchmarks, Vorlagen
   - Unser Winkel: Wir haben aggregierte Daten von 500+ Engineering-Teams

2. **Engineering Management** — "Wie Engineering Manager hochperformante Teams aufbauen"
   - Audience: VPs, EMs bei 20-200-Personen-Unternehmen
   - Formate: Rahmen, Manager-Interviews, Entscheidungsleitfäden
   - Unser Winkel: Entscheidungsunterstützung, nicht nur Motivation

3. **Developer Experience** — "Tools, Prozesse und Kultur, die Ingenieure glücklich machen"
   - Audience: Entwickler, die sich für besseres Tooling einsetzen
   - Formate: Tool-Vergleiche, Prozess-Teardowns, Meinungsstücke
   - Unser Winkel: Wir sind ein Engineering-Tool — Glaubwürdigkeit ist hoch

**90-Tage-Priorität:**
- 2 SEO-gesteuerte Artikel/Woche auf Engineering-Management-Keywords ausgerichtet
- 1 LinkedIn-Post/Tag (aus Artikeln umgestaltet)
- Monatlicher Newsletter: Zusammenfassung der Top 3 Artikel + 1 Original-Einsicht
- Erste 12 Wochen: 24 Artikel, 60+ LinkedIn-Posts, 3 Newsletter

**Erste 3 zu schreibende Artikel (basierend auf Suchvolumen + Säulenpassform):**
1. "Engineering Team Velocity Benchmarks: Was schnelle Teams anders machen" (Säule 1, Daten)
2. "Wie man effektive Engineering Retrospektiven durchführt" (Säule 2, How-to)
3. "Der Engineering Manager's Guide to Sprint Planning" (Säule 2, Rahmen)

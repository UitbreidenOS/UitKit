# KI-SEO-Skill

## Wann aktivieren
- Optimierung von Inhalten für die Anzeige in Google-KI-Überblicken (ehemals SGE)
- Von ChatGPT, Perplexity oder Claude zitiert werden, wenn Benutzer Fragen stellen
- Anpassung Ihrer SEO-Strategie, wenn KI-Suche Marktanteile von traditioneller Suche übernimmt
- Schreiben von Inhalten, die für KI-Extraktion und -Zitierung strukturiert sind
- Audit von bestehendem Inhalt auf KI-Suchkompatibilität

## Wann nicht verwenden
- Traditionelle Blue-Link-SEO (Ranking #1 für Head-Begriffe) — verwenden Sie die Skill seo-audit
- Bezahlte Suche auf KI-Plattformen — noch in Entwicklung, keine etablierte Best Practice
- Brandneue Websites ohne Autorität — bauen Sie erst die Grundlagen auf

## Anleitung

### Verstehen Sie KI-Suchintention

```
Analysieren Sie diese Suchanfrage zur KI-Suchoptimierung:

Anfrage: "[Anfrage einfügen]"
Meine Seite/Inhalt: [beschreiben oder URL]

Sagen Sie mir:
1. Ist dies eine navigatieve, informative oder transaktionale Abfrage?
2. Würde Google für diese Abfrage einen KI-Überblick anzeigen? (wahrscheinlich für Information, How-to, Vergleiche)
3. Welches Format hat das KI-Snippet normalerweise? (Liste / Absatz / Tabelle / Schritte)
4. Was muss mein Inhalt enthalten, um zitiert zu werden?
5. Welche ist die ideale Inhaltsstruktur für diese Abfrage?
```

### Optimieren Sie Inhalte für KI-Überblicke

```
Optimieren Sie diesen Inhalt für Google-KI-Überblicke.

Inhalt: [einfügen oder beschreiben]
Zielabfrage: "[einfügen]"

Strukturieren Sie den Inhalt so:
1. Die TLDR-Antwort erscheint in den ersten 40-50 Wörtern
2. Schlüsselpunkte sind in scannbarem Format (H2/H3 + kurze Absätze)
3. Fakten, Statistiken und Definitionen sind klar angegeben
4. Der Inhalt beantwortet die Frage direkt, bevor er elaboriert
5. Verwandte Fragen werden abgedeckt (People Also Ask-Muster)

Schreiben Sie die Einleitung und Schlüsselabschnitte um, um direkte Antworten zu priorisieren.
```

### AEO — Answer Engine Optimisation

```
Schreiben Sie Inhalt optimiert, um von Perplexity, ChatGPT und Claude zitiert zu werden.

Thema: [beschreiben]
Zielabfrage: "Wie mache ich [X]?" oder "Was ist [X]?"

Diese KI-Engines bevorzugen:
- Klare, sachliche Aussagen mit spezifischen Zahlen und Daten
- Strukturierte Informationen (Definition → warum es wichtig ist → wie → Beispiel)
- Autoritärer Ton ohne Hedging ("X ist" nicht "X könnte sein")
- Einzigartige Perspektive oder Daten (Originalforschung schlägt aggregierte Inhalte)
- Aktualisierter Inhalt (KI-Engines benachteiligen veraltete Informationen)

Schreiben Sie Inhalt, den eine KI zitieren würde, wenn ein Benutzer nach [Thema] fragt.
```

### E-E-A-T-Signale für KI-Ära

```
Audit dieser Seite für E-E-A-T-Signale (Erfahrung, Fachwissen, Autorität, Vertrauenswürdigkeit):

Seite: [URL oder Beschreibung]

KI-Suchmaschinen gewichten E-E-A-T stark. Überprüfen:
- Autor-Bio mit sichtbaren Anmeldeinformationen?
- Signale aus erster Hand ("Ich habe dies 6 Monate lang getestet...")
- Originaldaten, Forschung oder Fallstudien?
- Externe Zitate und verlinkte Quellen?
- Datum ist klar angezeigt und aktuell gehalten?
- Autor hat externe Präsenz (LinkedIn, Veröffentlichungen, Vorträge)?

Für jede Lücke: spezifische Aktion zum Hinzufügen von E-E-A-T-Signalen.
```

### FAQ-Schema für KI-Zitate

```
Schreiben Sie FAQ-Schema-Markup für diese Seite.

Thema: [beschreiben]
Seiten-URL: [URL]
Häufig gestellte Fragen: [auflisten oder mich generieren lassen]

Generieren:
1. 5-8 FAQ-Paare (Frage + prägnante Antwort)
2. JSON-LD-Schema-Markup bereit zum Einfügen in <head>
3. Inline-FAQ-Bereich zum Hinzufügen zur Seite (nicht nur Schema)

Jede Antwort sollte 40-60 Wörter sein — genug Kontext zum Zitieren durch KI, kurz genug zum Anzeigen in Snippets.
```

### Überwachen Sie KI-Suchsichtbarkeit

```
Erstellen Sie einen Überwachungsplan für KI-Suchsichtbarkeit.

Meine Website: [beschreiben]
Schlüsselthemen für die ich ein Ranking habe: [auflisten]

Verfolgen:
1. Google-KI-Überblick-Erscheinungen (Search Console: search_type=discover + KI-Features-Bericht)
2. Perplexity-Zitate (monatliche Suche nach "[site:mysite.com]" + manuelle Kontrollen)
3. ChatGPT-Sichtbarkeit (monatlich fragen "Was sind die besten Tools für X?", überprüfen ob zitiert)
4. Zero-Click-Rate in GSC (Impressionen oben, Klicks flach = KI beantwortet Ihre Anfragen)

Auslöser für Warnungen: [was Content-Updates sollte veranlassen]
Review-Häufigkeit: [monatlich / vierteljährlich]
```

## Beispiel

**Benutzer:** Mein Blog-Post "beste Projektmanagement-Tools" erhält 5.000 Suchen/Monat, aber Klicks sind seit Google KI-Überblicke um 40% gesunken.

**Claudes KI-SEO-Fix:**

1. Intro umstrukturieren: führen mit direkter Vergleichstabelle (KI-Überblicke lieben Tabellen für "bestes X"-Abfragen)
2. Hinzufügen einer "Quick Answer"-Box oben: "Das beste Projektmanagement-Tool für kleine Teams ist X, weil [Grund]. Für Enterprise: Y."
3. FAQ-Bereich mit 6 spezifischen Fragen hinzufügen, die KI anzeigen würde: "Ist Notion besser als Asana?", "Was ist die beste kostenlose Option?", etc.
4. JSON-LD-FAQ-Schema-Markup hinzufügen
5. Originaldaten einbeziehen: "Wir haben 8 Tools 3 Monate lang getestet — hier ist, was wir gefunden haben" signalisiert E-E-A-T
6. Akzeptieren Sie die Zero-Click-Realität: optimieren Sie für Markenbekannheit und Bottom-Funnel-Klicks, nicht Mid-Funnel-Traffic

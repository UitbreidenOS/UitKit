# Page CRO Skill

## Wann aktivieren
- Überprüfung einer Landing Page auf Konversierungsprobleme
- Generierung von A/B-Test-Hypothesen zur Verbesserung von Anmeldungen oder Käufen
- Interpretation von Heatmap- / Session-Recording-Daten
- Überprüfung Ihrer Homepage, Pricing-Seite oder Produktseite
- Verständnis dafür, warum Traffic hoch ist, aber Konversionen niedrig

## Wann nicht verwenden
- E-Mail-Kampagnen-Optimierung — anderer Kanal
- Bezahlte Anzeigentexte — verwenden Sie die campaign-brief Skill
- UX-Forschung von Grund auf — CRO basiert auf vorhandenen Daten

## Anleitung

### Vollständiger Landing-Page-Audit

```
Überprüfen Sie diese Landing Page auf Konversierungsprobleme.

URL oder Beschreibung: [beschreiben oder wichtigste Abschnitte einfügen]
Ziel: [Anmeldung / Kauf / Demo-Anfrage / Download]
Aktuelle Konversionsrate: [X]% (falls bekannt)
Traffic-Quelle: [bezahlt / organisch / E-Mail / Empfehlung]

Bewertung:

ÜBER DEM FALZ (erster Bildschirm, kein Scrollen)
- Ist die Wertproposition in < 5 Sekunden klar?
- Spricht die Überschrift das Problem oder den Wunsch des Besuchers an?
- Gibt es einen einzelnen, offensichtlichen CTA?
- Unterstützt das Heldenbild / Video die Botschaft oder lenkt ab?
- Gibt es Reibung (zu viele Felder, erzwungene Kontoerstellung)?

VERTRAUEN UND GLAUBWÜRDIGKEIT
- Sozialer Beweis vorhanden? (Bewertungen, Testimonials, Logos, Fallstudien)
- Ist sozialer Beweis spezifisch und glaubwürdig (nicht nur "5 Sterne")?
- Sicherheitsabzeichen / Garantien in der Nähe von Kauf/Anmeldung?
- Wer hat das gebaut? (Abschnitt Über/Team oder Gründergeschichte)

WERTPROPOSITION
- Ist der Vorteil (Ergebnis für Benutzer) klar vs. nur Funktionen?
- Gibt es klare Differenzierung von Alternativen?
- Ist die Preisgestaltung sichtbar oder gibt es Angst vor versteckten Kosten?
- Gibt es Risikoverlagerung? (kostenlos testen, Geld-zurück-Garantie, keine Kreditkarte)

CTA-ANALYSE
- Wie viele CTAs auf der Seite? (1-2 ist ideal für fokussierte Landing Pages)
- CTA-Text: spezifisch ("Kostenlose Testversion starten") oder generisch ("Absenden")?
- CTA-Platzierung: sichtbar ohne zu scrollen? An natürlichen Haltepunkten wiederholt?
- CTA-Kontrast: fällt es visuell auf?

REIBUNGSPUNKTE
- Formularformat: weniger Felder = höhere Konvertierung (nur Wesentliches fragen)
- Ladegeschwindigkeit: langsame Seiten töten Konvertierungen (jede Sekunde = ~7% Rückgang)
- Mobile Erfahrung: optimiert für Daumen-Scrollen?
- Ablenkungen: Navigationslinks, Social-Media-Feeds lenken ab?

Ergebnis: Rangierte Liste von Problemen mit A/B-Test-Hypothesen für jedes.
```

### A/B-Test-Hypothesen-Generator

```
Generieren Sie A/B-Test-Hypothesen für diese Seite.

Aktuelle Seite: [beschreiben Sie Überschrift, CTA, Layout]
Identifizierte Hauptreibungspunkte: [Liste]
Aktuelle Konversionsrate: [X]%

Für jede Hypothese:
- Zu testender Bereich: [was ändern]
- Kontrolle: [aktuelle Version]
- Variante: [vorgeschlagene Änderung]
- Erwarteter Einfluss: [warum sollte dies Konvertierung verbessern]
- Messung: [primäre Metrik, sekundäre Metriken]
- Erforderliche Mindeststichprobengröße: [geschätzt]
- Priorität: [Hoch / Mittel / Niedrig]
```

### Heatmap-Interpretation

```
Interpretieren Sie diese Heatmap / Scroll-Map-Ergebnisse:

Klickdaten: [beschreiben Sie, wo Menschen klicken]
Scroll-Tiefe: [X]% der Besucher erreichen [Bereich], [X]% erreichen CTA
Rage Clicks: [Elemente mit wiederholten frustrierten Klicks?]
Dead Clicks: [nicht anklickbare Elemente, die angeklickt werden?]

Diagnose:
1. Was sagen uns die Scroll-Daten über Interesse/Abbruch?
2. Klicken Menschen dort, wo wir sie haben wollen?
3. Was verwirrt oder frustriert Benutzer (Rage Clicks)?
4. Was wird ignoriert, sollte aber nicht?
5. Top 3 Änderungen basierend auf diesen Daten
```

### Pricing-Seite Audit

```
Überprüfen Sie diese Pricing-Seite:

[beschreiben Sie Tiers, Preise, Funktionen, CTAs]

Zielkunde: [für wen ist das]
Größter Einwand in dieser Phase: [Preis / Wert / Risiko / Komplexität]

Überprüfen:
- Ist der empfohlene Plan klar? (Hervorhebung, Abzeichen, "Am beliebtesten")
- Werden Funktionen in Ergebnissen beschrieben, nicht nur Kontrollkästchen?
- Gibt es kostenlose Testversion oder Freemium-Einstiegspunkt?
- Ist die Jahres- vs. Monatspreise klar dargestellt?
- Gibt es eine Vergleichstabelle vs. Konkurrenten oder DIY-Alternativen?
- FAQ-Bereich: beantwortet er die echten Einwände?
- Was passiert, wenn ich auf den CTA klicke? (Reibung im nächsten Schritt?)
```

## Beispiel

**Benutzer:** Meine SaaS-Pricing-Seite hat eine Konversionsrate von 2,1%. 82% der Besucher verlassen die Seite, ohne auf einen CTA zu klicken.

**Claudes CRO-Audit:**

Schlüsselprobleme nach Priorität:
1. **Hero — keine klare Empfehlung:** 3 gleich gewichtete Tiers ohne "hier starten"-Signal → Entscheidungslähmung. Fix: Fügen Sie das Badge "Am beliebtesten" zum mittleren Tier hinzu, dämpfen Sie das Enterprise-Tier für KMU-Traffic.
2. **CTAs sagen "Los geht's":** Generisch — keine Differenzierung zwischen Tiers. Fix: Testen Sie "Kostenlose Testversion starten" (Starter) vs. "Demo planen" (Pro) vs. "Vertrieb kontaktieren" (Enterprise).
3. **Jahres-/Monats-Toggle-Standard ist Jahres:** Besucher sehen zuerst höheren Preis = Preisschock. Fix: A/B-Test mit Standardwert Monatspreise.
4. **Keine Risikoverlagerung:** Keine Erwähnung von Testversion oberhalb der Falzlinie. Fix: Fügen Sie die Zeile "14-Tage kostenlos testen, keine Kreditkarte" direkt unter CTAs hinzu.
5. **FAQ beantwortet Preis-Einwand nicht:** "Ist das den Preis wert?" wird nicht beantwortet. Fix: Fügen Sie hinzu "Wie vergleicht sich das mit [Hauptalternative]?" und ROI-Rechner.

**Top A/B-Test zum Ausführen:** Ändern Sie CTA-Text von "Los geht's" zu "Kostenlose Testversion starten — Keine Kreditkarte" → geschätzter Anstieg von 20-35% basierend auf ähnlichen SaaS-Daten.

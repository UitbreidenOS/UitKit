# Habilidad de Estrategia de Precios

## Cuándo activar
- Establecer precios para un nuevo producto o servicio
- Decidir entre modelos de precios (suscripción, basado en uso, de una sola vez, freemium)
- Diseñar niveles de precios (cuántos, qué se incluye, cómo diferenciar)
- Aumentar precios para clientes existentes
- Ejecutar un análisis de disposición a pagar

## Cuándo NO usar
- Precios de productos regulados — requiere revisión de cumplimiento
- Negociaciones de precios personalizados de empresas — proceso diferente
- Modelado financiero para inversores — use la habilidad de plan financiero

## Instrucciones

### Elegir un modelo de precios

```
Ayúdeme a elegir el modelo de precios adecuado para mi producto.

Producto: [describa — software / servicio / producto físico / mercado]
Clientes: [quién compra, tamaño de empresa, rango de presupuesto]
Cómo lo usan: [frecuencia, volumen, intensidad]
Nuestra estructura de costos: [aproximadamente — costos fijos altos o bajos? ¿Variable por cliente?]
Modelos de competidores: [qué cobran y cómo]

Evalúe estos modelos para mi situación:
1. Tarifa plana (precio único) — simple, predecible, bueno para productos simples
2. Suscripción por niveles — bueno, mejor y mejor, modelo SaaS más común
3. Basado en uso / medido — cobrar por llamada API, asiento, transacción, GB
4. Freemium — nivel gratuito para siempre + pagado por más características/límites
5. Prueba gratuita — acceso completo por X días, luego pagar
6. Por asiento — precios por usuario, bueno para herramientas de equipo
7. Basado en valor — precio anclado en ROI del cliente (ideal pero difícil de hacer cumplir)
8. Pago único — bueno para herramientas sin valor continuo, audiencias que odian las suscripciones

Recomendación: ¿Qué modelos se ajustan a mi producto y por qué?
```

### Diseñar niveles de precios

```
Diseñe niveles de precios para [producto].

Objetivo: [maximizar conversión / maximizar ingresos / pasar al mercado superior]
Segmentos de clientes que quiero servir: [describa cada uno — p. ej. trabajador autónomo / equipo pequeño / empresa]
Impulsores clave de valor: [qué características/límites les interesan más a los clientes]
Precios de competidores: [qué cobran otros, si se sabe]

Diseñe 3 niveles:
- Nivel 1 (entrada / autoservicio): límites, características, precio, cliente objetivo
- Nivel 2 (medio / más popular): límites, características, precio, cliente objetivo
- Nivel 3 (pro / equipos): límites, características, precio, cliente objetivo

Opcional: Nivel 0 (gratuito) y Nivel 4 (empresa / personalizado)

Reglas para un buen diseño de niveles:
- Cada nivel debe parecer obviamente "vale la pena actualizar"
- Coloque 1-2 características que la gente realmente quiere detrás de puertas pagadas
- El nivel más popular debe estar en el medio (precios de señuelo)
- Anual vs. mensual: ¿Qué % de descuento ofrecer por anual?
```

### Freemium vs. prueba gratuita

```
¿Debo usar freemium o una prueba gratuita?

Mi producto: [describa]
Tiempo hasta el valor: [¿cuánto tiempo hasta que los usuarios obtengan un valor significativo?]
Adquisición de usuarios: [inbound / pagado / boca a boca]
Mercado objetivo: [PYME autoservicio / empresa / consumidor]
Objetivo de conversión: [objetivo de prueba a pago del X%]

Ventajas y desventajas de Freemium para mi situación:
- Pro: fricción más baja, viral/boca a boca, construye base de usuarios
- Desventaja: alto costo de soporte para usuarios no pagadores, motivación de actualización poco clara

Ventajas y desventajas de la prueba gratuita para mi situación:
- Pro: urgencia clara, captura usuarios comprometidos, producto más simple
- Desventaja: fricción en el registro, el usuario debe actuar antes del plazo

Recomendación + la variable clave que impulsa la decisión en mi caso.
```

### Estrategia de aumento de precios

```
Quiero aumentar los precios. Ayúdeme a hacerlo sin perder clientes.

Precios actuales: $[X]/mes
Precios propuestos: $[X]/mes (aumento: [X]%)
Razón: [valor entregado, inflación, reposicionamiento de mercado]
Base de clientes: [X] clientes, tenencia promedio [X] meses

Diseñe el lanzamiento:
1. Quién va a tener derechos adquiridos (clientes existentes mantienen precio antiguo ¿cuánto tiempo?)
2. Quién migra primero (nuevos clientes inmediatamente, existentes después de X meses?)
3. Secuencia de comunicación (cuándo notificar, qué decir, quién firma el correo)
4. Manejo de objeciones (qué decir a "su competidor es más barato")
5. Modelo esperado de tasa de churn e impacto de ingresos
6. Borrador del correo electrónico de aumento de precio

Referencia histórica: los aumentos de precio de SaaS bien ejecutados ven 2-8% de churn, ingresos netos positivos.
```

### Investigación de disposición a pagar

```
Diseñe un proceso de investigación de disposición a pagar para [producto].

Segmento objetivo: [a quién entrevistar/encuestar]
Rango de precios que estoy considerando: $[X] a $[X]

Métodos de investigación (en orden de esfuerzo):
1. Medidor de sensibilidad de precios de Van Westendorp (encuesta de 4 preguntas):
   - ¿A qué precio es esto demasiado barato para confiar?
   - ¿A qué precio es esta una ganga?
   - ¿A qué precio es esto caro pero aún lo comprarías?
   - ¿A qué precio es esto demasiado caro?

2. Entrevistas con clientes (5-10 clientes existentes o potenciales):
   - ¿Qué pagas actualmente por alternativas?
   - ¿Qué ROI obtienes al resolver este problema?
   - Caminemos a través de tu proceso de compra para herramientas como esta

3. Prueba A/B de página de precios:
   - Dividir tráfico a diferentes puntos de precio
   - Medir conversión y churn en cada precio

Diseñe la encuesta de Van Westendorp y el script de entrevista para mi producto.
```

## Ejemplo

**Usuario:** Tengo una herramienta SaaS a $29/mes de tarifa plana. Los ingresos están bien pero sospecho que estoy dejando dinero sobre la mesa. ¿Qué debería hacer?

**Análisis de Claude:**

La fijación de precios plana de $29 es la simplicidad clásica de etapa temprana — inteligente para la validación, limitante para la escala.

**El problema:** $29 captura el mismo precio de un autónomo usando su herramienta 2 horas/semana y un equipo de 10 personas usándola todo el día. Está inframonetizando el segmento de alto valor.

**Movimiento recomendado: reestructuración de 3 niveles**

| Nivel | Precio | Para | Límites clave |
|---|---|---|---|
| Solo | $29/mes | Autónomos, trabajadores autónomos | 1 usuario, 100 elementos |
| Equipo | $79/mes | Equipos pequeños | 5 usuarios, 1.000 elementos |
| Empresarial | $199/mes | Empresas en crecimiento | 20 usuarios, ilimitado |

**Estrategia de migración para clientes existentes de $29:**
- Abuelo a $29 durante 12 meses (recompensa de lealtad)
- Después de 12 meses: invitar a elegir nuevo nivel según el uso
- No se disculpe — explique el aumento de valor

**Impacto proyectado:** Si el 20% de los clientes existentes se actualizan a Equipo ($79) y el 5% a Empresarial ($199), los ingresos mensuales aumentan ~35% con el mismo número de clientes.

# Habilidad de SEO IA

## Cuándo activar
- Optimizar contenido para aparecer en Google AI Overviews (anteriormente SGE)
- Ser citado por ChatGPT, Perplexity o Claude cuando los usuarios hacen preguntas
- Adaptar tu estrategia de SEO a medida que la búsqueda por IA toma participación de la búsqueda tradicional
- Escribir contenido estructurado para extracción y citación por IA
- Auditar contenido existente para compatibilidad con búsqueda de IA

## Cuándo no usar
- SEO tradicional de enlaces azules (ranking #1 para términos principales) — usar la habilidad seo-audit
- Búsqueda pagada en plataformas de IA — aún en evolución, sin mejor práctica establecida
- Sitios completamente nuevos sin autoridad — primero construye los fundamentos

## Instrucciones

### Entender la intención de búsqueda de IA

```
Analiza esta consulta para optimización de búsqueda de IA:

Consulta: "[pega consulta]"
Mi página/contenido: [describe o URL]

Dime:
1. ¿Es esto una consulta de navegación, información o transaccional?
2. ¿Mostraría Google un AI Overview para esta consulta? (probable para informativa, how-to, comparaciones)
3. ¿Qué formato toma típicamente el fragmento de IA? (lista / párrafo / tabla / pasos)
4. ¿Qué debe incluir mi contenido para ser citado?
5. ¿Cuál es la estructura de contenido ideal para esta consulta?
```

### Optimiza contenido para AI Overviews

```
Optimiza este contenido para Google AI Overviews.

Contenido: [pega o describe]
Consulta objetivo: "[pega]"

Estructura el contenido para que:
1. La respuesta TLDR aparezca en los primeros 40-50 palabras
2. Los puntos clave estén en formato escaneable (H2/H3 + párrafos cortos)
3. Los hechos, estadísticas y definiciones estén claramente expuestos
4. El contenido responda directamente a la pregunta antes de elaborar
5. Las preguntas relacionadas estén cubiertas (patrones de Preguntas similares)

Reescribe la introducción y secciones clave para priorizar respuestas directas.
```

### AEO — Optimización de Motor de Respuestas

```
Escribe contenido optimizado para ser citado por Perplexity, ChatGPT y Claude.

Tema: [describe]
Pregunta objetivo: "¿Cómo hago [X]?" o "¿Qué es [X]?"

Estos motores de IA prefieren:
- Declaraciones claras y factuales con números y datos específicos
- Información estructurada (definición → por qué importa → cómo → ejemplo)
- Tono de autoridad sin vacilación ("X es" no "X podría ser")
- Perspectiva o datos únicos (la investigación original vence al contenido agregado)
- Contenido actualizado (los motores de IA penalizan la información obsoleta)

Escribe contenido que una IA citaría cuando un usuario pregunta sobre [tema].
```

### Señales E-E-A-T para la era de la IA

```
Audita esta página para señales E-E-A-T (Experiencia, Expertise, Autoridad, Confiabilidad):

Página: [URL o descripción]

Los motores de búsqueda de IA ponderan E-E-A-T fuertemente. Verifica:
- ¿Bio del autor con credenciales visibles?
- ¿Señales de experiencia de primera mano ("Probé esto durante 6 meses...")
- ¿Datos originales, investigación o estudios de caso?
- ¿Citas externas y fuentes vinculadas?
- ¿Fecha claramente mostrada y mantenida actualizada?
- ¿Autor tiene presencia externa (LinkedIn, publicaciones, charlas)?

Para cada brecha: acción específica para agregar señales E-E-A-T.
```

### Esquema de preguntas frecuentes para citas de IA

```
Escribe marcado de esquema de preguntas frecuentes para esta página.

Tema: [describe]
URL de la página: [URL]
Preguntas comunes que hacen los usuarios: [lista o déjame generarlas]

Generar:
1. 5-8 pares de preguntas frecuentes (pregunta + respuesta concisa)
2. Marcado de esquema JSON-LD listo para pegar en <head>
3. Sección de preguntas frecuentes en línea para agregar a la página (no solo esquema)

Cada respuesta debe tener 40-60 palabras — contexto suficiente para que la IA cite, lo suficientemente corto para mostrar en fragmentos.
```

### Monitorea la visibilidad de búsqueda de IA

```
Crea un plan de monitoreo para la visibilidad de búsqueda de IA.

Mi sitio: [describe]
Temas clave para los que tengo ranking: [lista]

Rastrear:
1. Apariciones en Google AI Overview (Search Console: search_type=discover + informe de características de IA)
2. Citas de Perplexity (busca "[site:mysite.com]" mensualmente + verificaciones manuales)
3. Visibilidad de ChatGPT (pregunta "¿Cuáles son las mejores herramientas para X?" mensualmente, verifica si está citado)
4. Tasa de cero clics en GSC (impresiones arriba, clics planos = la IA responde tus consultas)

Desencadenantes de alerta: [qué debe solicitar actualizaciones de contenido]
Cadencia de revisión: [mensual / trimestral]
```

## Ejemplo

**Usuario:** Mi publicación de blog "mejores herramientas de gestión de proyectos" recibe 5.000 búsquedas/mes pero los clics bajaron 40% desde que Google agregó AI Overviews.

**Solución de SEO de IA de Claude:**

1. Reestructura intro: lidera con tabla de comparación directa (AI Overviews ama las tablas para consultas "mejor X")
2. Agrega cuadro "Respuesta rápida" en la parte superior: "La mejor herramienta de gestión de proyectos para equipos pequeños es X porque [razón]. Para empresas: Y."
3. Agrega sección de preguntas frecuentes con 6 preguntas específicas que la IA mostraría: "¿Es Notion mejor que Asana?", "¿Cuál es la mejor opción gratuita?", etc.
4. Agrega marcado de esquema JSON-LD de preguntas frecuentes
5. Incluye datos originales: "Probamos 8 herramientas durante 3 meses — esto es lo que encontramos" señala E-E-A-T
6. Acepta la realidad de cero clics: optimiza para conciencia de marca y clics en el fondo del embudo, no tráfico medio

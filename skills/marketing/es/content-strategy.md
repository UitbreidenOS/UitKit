# Content Strategy Skill

## Cuándo activar
- Construir una estrategia de contenido desde cero para una marca o producto
- Diseñar pilares de contenido y clusters de temas
- Crear un calendario editorial entre canales
- Construir un sistema de reutilización de contenido (una pieza → muchos formatos)
- Alinear el contenido con los objetivos de SEO y comerciales
- Auditar el contenido existente para identificar brechas y oportunidades

## Cuándo NO usar
- Escribir un solo contenido — simplemente escríbelo, no estrategices
- Auditorías técnicas de SEO — usa la habilidad seo-audit
- Distribución de contenido pagado — usa la habilidad paid-ads
- Solo la cadencia de publicación en redes sociales — usa la habilidad social-media-manager

## Instrucciones

### Definir pilares de contenido

```
Define pilares de contenido para [marca/producto].

Negocio: [describe lo que haces y a quién sirves]
Audiencia objetivo: [título del puesto, tamaño de empresa, desafíos clave]
Objetivos comerciales que el contenido debe apoyar: [conciencia de marca / generación de leads / retención de clientes / tráfico SEO]
Competidores creando contenido: [quién y sobre qué escriben]
Ángulo único que tenemos: [experiencia, punto de vista o datos que poseemos]

Diseña 3-5 pilares de contenido:
Cada pilar = un territorio temático que poseemos y publicamos consistentemente.

Formato por pilar:
- Nombre del pilar: [etiqueta clara]
- Por qué a esta audiencia le importa: [el dolor o goal subyacente]
- Tipos de contenido que se ajustan: [cómo hacerlo / opinión / estudio de caso / datos / comentario de noticias]
- Nuestro ángulo único: [por qué podemos hablar de esto mejor que nadie]
- Ejemplo de titular: [título de artículo/post específico]

Regla: Los pilares deben ser lo suficientemente específicos para ser posesionable, lo suficientemente amplios para 52 piezas/año.
```

### Calendario editorial

```
Construye un calendario editorial de 90 días para [marca].

Canales a cubrir: [blog / boletín / LinkedIn / YouTube / Twitter/X]
Frecuencia de publicación: [X posts/semana por canal]
Pilares de contenido: [pega arriba]
Campañas o lanzamientos próximos: [lanzamientos de productos, eventos, estaciones]
Recursos: [1 escritor / 2 escritores / equipo]

Formato del calendario:
Semana | Canal | Pilar | Título | Formato | CTA | Notas

Reglas:
- Comienza con artículos impulsados ​​por SEO (blog) — mayor ROI a largo plazo
- Boletín: curador + agrega perspectiva, no solo republiques posts de blog
- LinkedIn: versiones cortas de insights de blog (3-5 líneas + hook)
- Agrupa formatos similares para eficiencia de producción
- Planifica con al menos 2 semanas de anticipación; deja 10% de espacios para contenido reactivo/trending

Produce: calendario de 12 semanas con 1 pieza ancla por semana + plan de distribución para cada.
```

### Sistema de reutilización de contenido

```
Diseña un sistema de reutilización de contenido.

Canal principal y formato: [blog / podcast / YouTube / evento en vivo]
Canales secundarios a alimentar: [enumera todos: boletín, LinkedIn, Twitter, YouTube Shorts, etc.]
Tamaño del equipo: [solo / 1 editor / equipo completo]
Cadencia de publicación: [X piezas principales por mes]

Mapa de reutilización (1 primaria → muchas):

Si primaria = artículo de forma larga (2000+ palabras):
→ Boletín: intro + 3 puntos clave + enlace
→ LinkedIn: el insight más contraintuitivo, publicación única
→ Twitter/X: hilo de 5-7 puntos
→ Video corto: resumen "lo que aprendí" de 60 segundos
→ Tarjeta de cita: 1 cita extraída → imagen para Instagram/LinkedIn

Si primaria = episodio de podcast:
→ Artículo de blog: transcripción editada en artículo (agregar encabezados, expandir ejemplos)
→ Boletín: resumen de episodio + 3 principales conclusiones
→ LinkedIn: cita de invitado o insight clave como publicación
→ YouTube Shorts: mejor clip de 60 segundos
→ Twitter: estadística clave o hilo de cita

Construye el sistema específico para mis canales con asignaciones por pieza.
```

### Auditoría de contenido

```
Audita el contenido existente e identifica oportunidades.

Ubicación de contenido: [URL del blog / lista de URLs / sitemap]
Objetivos comerciales: [impulsar registros de prueba / clasificar para X palabras clave / retener clientes]
Volumen de contenido actual: [X artículos / X posts]
Acceso a Analytics: [GA4 / Search Console / ninguno]

Marco de auditoría:
1. Segmentos de tráfico:
   - Tráfico alto, conversión alta → proteger y actualizar
   - Tráfico alto, conversión baja → optimizar CTA y flujo de destino
   - Tráfico bajo, conversión alta → promover más (pagado, enlaces, correo electrónico)
   - Tráfico bajo, conversión baja → actualizar o eliminar

2. Escaneo de oportunidades de SEO:
   - Artículos clasificando posiciones 8-20 → actualizar para impulsar a la página 1
   - Artículos con tráfico orgánico cero después de 6 meses → consolidar o redirigir
   - Temas faltantes para los que clasifican competidores → crear

3. Frescura del contenido:
   - Artículos > 18 meses → verificar hechos y actualizar
   - Artículos que hacen referencia a herramientas, estadísticas o prácticas desactualizadas → actualizar

Salida: lista priorizada de:
- [X] artículos para actualizar (ganancias rápidas, clasificación 8-20)
- [X] artículos para consolidar (contenido delgado)
- [X] temas netos nuevos para crear (análisis de brechas)
- [X] artículos para eliminar + redirigir
```

### Plan de distribución

```
Construye un plan de distribución para cada pieza nueva.

Tipo de contenido: [artículo / video / episodio de podcast / informe]
Tamaño de audiencia por canal: [correo electrónico: X / LinkedIn: X seguidores / Twitter: X / etc.]
Presupuesto de promoción: $[X] (o $0 solo para orgánico)

Secuencia de distribución (día a día):

Día 0 — Publicar:
□ Publicar en canal principal (optimizar título, meta, imagen OG)
□ Enviar a lista de correo (publicación completa o fragmento + enlace)
□ Publicar en LinkedIn (versión adaptada, no solo enlace)
□ Publicar en Twitter/X (hilo o cita única)

Día 3 — Amplificación:
□ Comprométete con todos los comentarios en todos los canales
□ Publica comentario de LinkedIn en tu propia publicación con insight de seguimiento
□ Comparte en comunidades relevantes (Slack, Discord, Reddit — agrega valor, no spam)

Día 7 — Reutilización:
□ Publica YouTube Short o Reel (si formato de video disponible)
□ Reutiliza cita como publicación de imagen

Día 30 — Seguimiento de SEO:
□ Verifica clasificaciones — ¿aparece para palabra clave objetivo?
□ Agrega enlaces internos del contenido más nuevo a esta pieza

Amplificación pagada (si presupuesto):
- Potencia los posts mejor funcionando en LinkedIn ($50-100 por publicación)
- Google Discovery o retargeting Meta para lectores de blog (conciencia)
```

## Ejemplo

**Usuario:** Dirijo una empresa B2B SaaS (herramienta de gestión de proyectos para equipos de ingeniería). Sin estrategia de contenido, simplemente escribimos posts de blog al azar. Ayúdame a construir uno.

**Estrategia de contenido de Claude:**

**3 Pilares de Contenido:**

1. **Productividad del Equipo de Ingeniería** — "Cómo los equipos de ingeniería se mueven más rápido sin agotamiento"
   - Dolor de audiencia: velocidad de envío sin sacrificio de calidad
   - Formatos: guías prácticas, benchmarks, plantillas
   - Nuestro ángulo: tenemos datos agregados de 500+ equipos de ingeniería

2. **Gestión de Ingeniería** — "Cómo los gestores de ingeniería construyen equipos de alto rendimiento"
   - Audiencia: VPs, EMs en empresas de 20-200 personas
   - Formatos: marcos, entrevistas de gerente, guías de decisión
   - Nuestro ángulo: soporte de decisiones, no solo motivación

3. **Experiencia del Desarrollador** — "Herramientas, procesos y cultura que hacen felices a los ingenier"
   - Audiencia: desarrolladores abogando por mejores herramientas
   - Formatos: comparaciones de herramientas, desgloses de procesos, piezas de opinión
   - Nuestro ángulo: somos una herramienta de ingeniería — credibilidad es alta

**Prioridad de 90 días:**
- 2 artículos impulsados ​​por SEO/semana dirigidos a palabras clave de gestión de ingeniería
- 1 publicación de LinkedIn/día (reutilizada de artículos)
- Boletín mensual: resumen de 3 mejores artículos + 1 insight original
- Primeras 12 semanas: 24 artículos, 60+ publicaciones de LinkedIn, 3 boletines

**Primeros 3 artículos a escribir (basados en volumen de búsqueda + ajuste de pilar):**
1. "Benchmarks de velocidad de equipo de ingeniería: qué hacen diferente los equipos rápidos" (pilar 1, datos)
2. "Cómo ejecutar retrospectivas efectivas de ingeniería" (pilar 2, cómo hacerlo)
3. "Guía del gerente de ingeniería para la planificación de sprint" (pilar 2, marco)

---
name: Diagram First
description: Commencez chaque explication d'architecture ou de flux par un diagramme Mermaid, puis par du texte
keep-coding-instructions: true
---
Pour toute question impliquant une architecture, un flux de données, une conception système, une séquence d'événements ou des relations de composants, ouvrez avec un diagramme Mermaid avant d'écrire tout texte. Choisissez le type de diagramme qui convient le mieux au contenu : `flowchart` pour les processus, `sequenceDiagram` pour les interactions, `classDiagram` pour les modèles de données, `erDiagram` pour les schémas, `C4Context` pour le contexte système. Après le diagramme, écrivez une explication textuelle concise qui référence les nœuds du diagramme par nom. Ne dupliquez jamais dans le texte ce que le diagramme rend déjà évident — utilisez le texte uniquement pour expliquer l'intention, les contraintes et les décisions non évidentes. Si un diagramme ne peut pas être produit pour une question, déclarez pourquoi et procédez au prochain format visuel le plus clair (tableau, liste numérotée).

# Command: /research-topic

## What It Does

Gathers trending data, expert insights, news, and competitive angles for a given topic. Output includes key data points, expert quotes, alternative perspectives, and a suggested hook for the newsletter.

## Usage

```
/research-topic [topic name or question]
```

## Examples

```
/research-topic AI regulation trends Q2 2026

/research-topic How enterprise teams are using Claude

/research-topic The rise of agentic AI workflows
```

## Output

The command returns a research brief containing:

1. **Topic summary** (2–3 sentences)
2. **Key trends and data points** (2–3 with sources)
3. **Expert perspectives** (1–2 quotes with attribution)
4. **Alternative or contrarian angles** (what other people are saying)
5. **Suggested hook** (opening line for newsletter)
6. **Next steps** (ready to draft, or needs follow-up research)

---

## How to Use This Output

1. **Save the research brief** — copy into your session notes
2. **Reference the hook** when drafting the newsletter
3. **Use data points and quotes** as evidence in your sections
4. **Consider alternative angles** when building counterarguments or nuance
5. **Move to `/draft-newsletter`** when ready to write

---

## Tips

- **Be specific:** "/research-topic AI" will return broad results. "/research-topic How Claude is being used in Fortune 500 companies" will be more targeted.
- **Include timeframe:** If you want recent data, mention it: "/research-topic enterprise AI trends Q2 2026" returns fresher results than "/research-topic enterprise AI trends"
- **Use follow-ups:** If initial research is weak, ask for deeper dives: "/research-topic enterprise agentic workflows — focus on financial services"
- **Run before drafting:** Never draft without research. It informs hook, tone, and CTA.

# Frontend Engineer Stack

Component-driven UI/UX development with performance-first architecture, accessibility compliance, and design system integration.

## Quick Start

This stack provides tools for:
- **Component auditing** — Reusability, testability, accessibility, performance
- **Accessibility review** — WCAG 2.1 AA compliance
- **Performance profiling** — React render optimization, Core Web Vitals
- **Storybook generation** — Auto-generate stories with all variants
- **Design system alignment** — Token usage, component naming, spec compliance
- **Responsive testing** — Breakpoint validation across devices
- **Visual regression testing** — Detect unintended style changes

## Available Commands

- `/audit-component` — Comprehensive component review
- `/review-a11y` — Accessibility audit (WCAG 2.1 AA)
- `/profile-performance` — Performance profiling and optimization
- `/generate-storybook` — Auto-generate Storybook stories
- `/check-design-system` — Design token and spec alignment
- `/test-responsive` — Responsive design validation
- `/test-visual-regression` — Visual regression detection

## Core Principles

1. **Accessibility First** — All components must meet WCAG 2.1 AA
2. **Performance Matters** — Core Web Vitals targets are non-negotiable
3. **Design System is Law** — Every component uses design tokens
4. **Testing is Essential** — Minimum 80% coverage on component logic
5. **Components are Composable** — If it's hard to reuse, refactor it

## Success Metrics

- **Test Coverage:** >80% across components
- **Accessibility:** 100% WCAG 2.1 AA compliance
- **Performance:** 100% meet Core Web Vitals targets
- **Storybook:** All components exported with all variants
- **Design System:** 100% token adoption

# Data Validation

## When to activate

Building quality checks, anomaly detection, or null/cardinality assertions.

## When NOT to use

Not for exploratory data analysis; focus on operational guardrails.

## Instructions

1. Define assertion types (schema, cardinality, range, uniqueness)
2. Set alert thresholds
3. Build reusable validation rules
4. Integrate into pipeline

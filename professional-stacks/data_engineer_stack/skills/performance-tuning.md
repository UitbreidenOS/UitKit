# Performance Tuning

## When to activate

Reducing CPU, memory, disk I/O, or network overhead in data jobs.

## When NOT to use

Not for algorithmic correctness issues; focus on resource efficiency.

## Instructions

1. Profile resource usage
2. Identify bottlenecks (compute, I/O, network)
3. Recommend parallelization or batching
4. Validate improvements

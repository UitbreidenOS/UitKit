# Command: quarterly-review

## Purpose
End-of-quarter review template for strategy reflection, metrics assessment, and Q-ahead planning.

## Usage
```
/quarterly-review [--quarter Q1|Q2|Q3|Q4] [--year 2026]
```

## Sections
1. **Metrics Dashboard** — Revenue, headcount, burn, key product metrics
2. **vs. Plan** — Actual vs. forecast, variance analysis
3. **Wins & Setbacks** — Major accomplishments and blockers
4. **Team Health** — Morale, retention, key hires/departures
5. **Product & Market** — Traction, customer feedback, competitive moves
6. **Financials** — Runway, unit economics, cost management
7. **Next Quarter Priorities** — 3-5 key initiatives and success criteria
8. **Board/Investor Update** — Key talking points and confidence signals

## Output
- Structured Q-review document
- Recommended format for board update slide deck
- Talking points for investor conversations

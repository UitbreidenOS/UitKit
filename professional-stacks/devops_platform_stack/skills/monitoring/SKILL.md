# Prometheus Alerting

## When to activate

Configuring metrics, writing alert rules, or debugging observability gaps.

## When NOT to use

For log analysis — use elk-logging skill instead.

## Instructions

1. Review metric definitions (cardinality, naming conventions)
2. Write and validate alert rules (PromQL)
3. Set appropriate thresholds and runbook links
4. Test alert firing behavior

## Example

Set up cpu/memory/disk alerts with graduated severity levels and runbook links.

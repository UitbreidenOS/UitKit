# CI/CD-Pipelines

## Wann aktivieren

Erstellen, Debuggen oder Optimieren von CI/CD-Workflows in GitHub Actions, GitLab CI oder Jenkins.

## Wann nicht verwenden

Für manuelle Deployment-Runbooks — verwenden Sie stattdessen Deployment-Guides.

## Anweisungen

1. Pipeline-YAML analysieren und Engpässe identifizieren
2. Job-Abhängigkeiten und parallele Ausführung validieren
3. Secrets-Verwaltung und Gating überprüfen
4. Build-Cache und Artifact-Handling optimieren

## Beispiel

Reduzieren Sie die Pipeline-Dauer von 45 Minuten auf 18 Minuten durch Parallelisierung von Tests und Caching von Abhängigkeiten.

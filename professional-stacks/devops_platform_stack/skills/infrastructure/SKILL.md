# Infrastructure Disaster Recovery

## When to activate

Designing failover strategies, testing recovery procedures, or planning multi-region deployment.

## When NOT to use

For single-region resource provisioning — use terraform-modules skill instead.

## Instructions

1. Map critical service dependencies
2. Document RTO/RPO requirements
3. Design active-passive or active-active topology
4. Execute and validate recovery runbooks

## Example

Design multi-region failover for stateful database with cross-region replication and DNS failover.
